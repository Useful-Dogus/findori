# Feature Specification: 카드 뉴스 시각 언어 시스템 구축

## Overview

이 기능의 핵심 목표는 **카드 뉴스에 일관된 시각 언어와 고정 레이아웃을 부여하는 것**이다. 두 가지 축으로 구성된다:

1. **시각 언어 구축**: 풍부한 이미지 라이브러리를 통해 독자가 이미지만 보고도 1초 안에 카드의 맥락을 이해할 수 있도록 한다
2. **고정 카드 크기**: 모든 카드를 동일한 종횡비로 통일하여, 독자가 카드를 넘길 때 레이아웃 변동(스크롤 점프)이 없는 매끄러운 탐색 경험을 제공한다

이미지는 세 가지 역할을 수행한다:
1. **정보 전달**: 주가 상승/하락, 실적, 환율 등 금융 테마를 즉시 전달
2. **개념 설명**: 추상적 투자 개념(리스크, 기회, 사이클)을 직관적 비유로 표현
3. **감정 유도**: 독자의 공감, FOMO, 불안, 기대를 자극해 콘텐츠 소비를 촉진

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 이미지로 카드 맥락 즉시 파악 (Priority: P1)

독자가 카드를 열람할 때, 텍스트를 읽기 전에 이미지만으로 해당 카드의 성격(상승/하락/경고/기회 등)을 즉시 파악한다. 삼성전자 실적 카드에는 삼성을 연상시키는 이미지가, 환율 급등 카드에는 환율을 상징하는 이미지가 표시되어 정보 전달 속도가 높아진다.

**Why this priority**: 시각 언어의 가장 직접적인 사용자 가치. 이미지가 없거나 맥락에 맞지 않으면 이 기능 전체의 의의가 없다.

**Independent Test**: 이미지 라이브러리에서 각 카테고리 키별로 카드를 생성하여, 이미지가 카드 내용과 주제적으로 일치하는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 삼성전자 실적 관련 카드가 있을 때, **When** 독자가 카드를 열람하면, **Then** 기업 또는 실적을 연상시키는 이미지가 표시된다
2. **Given** 원달러 환율 급등 관련 카드가 있을 때, **When** 독자가 카드를 열람하면, **Then** 화폐/환율을 상징하는 이미지가 표시된다
3. **Given** AI가 이미지 라이브러리의 어떤 키도 매핑하지 못하는 뉴스를 처리할 때, **When** 카드가 생성되면, **Then** 기본 이미지가 표시되며 렌더링 오류는 발생하지 않는다

---

### User Story 2 - 감정·공감 이미지로 독자 몰입 유도 (Priority: P2)

개인 투자자의 심리(불안, FOMO, 결정의 순간, 웃음)를 담은 이미지를 통해 독자가 카드 내용에 감정적으로 공감하고 몰입한다. 어려운 금융 정보를 인간적인 맥락으로 연결하여 콘텐츠 소비율과 공유율을 높인다.

**Why this priority**: 정보 전달(P1) 다음으로 중요한 독자 경험 요소. 유머와 공감 이미지는 경쟁 카드 뉴스와 차별화되는 핵심 요인이다.

**Independent Test**: 감정 카테고리 이미지(불안, FOMO, 결정 순간, 유머)가 라이브러리에 포함되어 있고, AI가 적절한 카드에 배정하는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 시장 급락 관련 카드가 있을 때, **When** 독자가 해당 카드를 열람하면, **Then** 투자자의 불안·공포를 표현한 이미지 또는 적절한 유머 이미지가 표시된다
2. **Given** 급등주 관련 카드가 있을 때, **When** 독자가 해당 카드를 열람하면, **Then** FOMO나 기대감을 자극하는 이미지가 표시된다
3. **Given** 감정 카테고리 이미지가 라이브러리에 포함된 상태에서 AI가 감정적 맥락의 카드를 생성할 때, **When** 이미지 키가 선택되면, **Then** 해당 이미지가 과도한 공포 조장 없이 금융 콘텐츠 맥락에 맞게 표시된다

---

### User Story 3 - 기업·환경 이미지로 현실감 강화 (Priority: P2)

주요 기업의 대표 이미지(로고, 건물, 공장), 한국·미국 증권거래소 이미지, 도심 금융가 사진 등 실사 이미지를 통해 카드 뉴스의 현실감과 신뢰도가 높아진다. 독자는 추상적 정보가 아닌 실제 맥락 속의 정보로 받아들인다.

**Why this priority**: 브랜드 신뢰도와 콘텐츠 품질을 높이는 요소. 특히 기업 관련 이슈에서 회사 이미지 매핑은 차별화된 가치를 제공한다.

**Independent Test**: 주요 기업 관련 카드와 거래소/시장 환경 카드에서 각각 대응 이미지가 표시되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 애플·엔비디아·삼성 등 주요 기업 관련 카드가 있을 때, **When** 독자가 카드를 열람하면, **Then** 해당 기업을 연상시키는 실사 이미지가 표시된다
2. **Given** 한국 증시 관련 카드가 있을 때, **When** 독자가 카드를 열람하면, **Then** 한국거래소(KRX) 또는 여의도 금융가 이미지가 표시된다
3. **Given** 인지도가 낮은 기업 관련 카드가 있을 때, **When** 독자가 카드를 열람하면, **Then** 해당 기업 전용 이미지 대신 업종 또는 상황에 맞는 일반 기업/건물 이미지가 표시된다

---

### User Story 4 - 카드 간 일정한 크기로 매끄러운 탐색 (Priority: P1)

독자가 카드 뉴스를 좌우로 넘길 때 카드마다 높이가 달라 스크롤 위치가 튀는 현상이 없어진다. 모든 카드가 동일한 종횡비로 고정되어 있어, 화면을 전환해도 레이아웃이 안정적으로 유지된다.

**Why this priority**: 현재 카드 높이 불일치는 사용자 경험을 직접적으로 해치는 버그성 이슈다. 이미지 도입과 함께 해결하지 않으면 이미지를 추가해도 레이아웃 불안정이 지속된다.

**Independent Test**: 카드 타입이 서로 다른 카드(cover, stats, cause 등)를 연속으로 탐색할 때 카드 간 높이 변화가 없는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 여러 타입의 카드(텍스트 많은 카드, 짧은 카드)가 혼합된 이슈가 있을 때, **When** 독자가 카드를 순서대로 넘기면, **Then** 모든 카드가 동일한 크기로 표시되어 스크롤 위치 변화가 없다
2. **Given** 카드 내용이 고정 크기를 초과하는 텍스트를 포함할 때, **When** 카드가 렌더링되면, **Then** 텍스트가 고정 크기 안에서 적절히 처리되어(말줄임·스크롤 등) 카드 외부 크기는 유지된다
3. **Given** 모바일과 데스크탑 화면 크기에서 카드를 열람할 때, **When** 카드가 표시되면, **Then** 화면 크기에 맞게 반응형으로 조정되되 모든 카드의 종횡비는 동일하게 유지된다

---

### User Story 5 - AI의 풍부한 이미지 키 선택 (Priority: P3)

AI가 카드를 생성할 때 단순한 8개 테마 카테고리뿐만 아니라, 감정·환경·기업·상징 등 다차원 이미지 라이브러리에서 해당 카드 내용에 가장 적합한 이미지 키를 선택한다. 이를 통해 카드별로 차별화된 시각 표현이 가능해진다.

**Why this priority**: P1-P3의 효과를 극대화하는 AI 활용 요소. 라이브러리가 구축되어야 의미 있는 선택이 가능하다.

**Independent Test**: 다양한 유형의 뉴스(기업 실적, 환율, 시장 급락, AI 섹터 등)에 대해 AI가 선택한 이미지 키가 해당 카드 내용과 관련성이 높은지 검토한다.

**Acceptance Scenarios**:

1. **Given** AI가 엔비디아 AI 칩 수요 관련 카드를 생성할 때, **When** 이미지 키를 선택하면, **Then** `ai-chip` 또는 해당 기업 이미지 키가 선택된다
2. **Given** AI가 한국 코스피 하락 관련 카드를 생성할 때, **When** 이미지 키를 선택하면, **Then** `stock-down` 또는 거래소 환경 이미지 키가 선택된다
3. **Given** AI가 100장의 카드를 생성할 때, **When** 이미지 키 선택 결과를 검토하면, **Then** 이미지 라이브러리 외 키 비율이 2% 이하이다

---

### Edge Cases

- 특정 기업의 이미지를 라이브러리에서 찾지 못할 때, AI가 올바른 업종/상황 대체 키를 선택하는가?
- 하나의 카드에 여러 테마(예: 삼성 실적 + 원화 환율)가 섞인 경우, AI는 어떤 기준으로 이미지 키를 선택하는가?
- 라이브러리에 이미지 파일이 누락(URL 404)된 경우 카드가 깨지지 않고 폴백되는가?
- 이미지 라이브러리에 새 키를 추가할 때, 기존 카드(이미 저장된 DB)의 이미지는 그대로 유지되는가?
- 기존 `CardVisual` (gradient only) 카드는 이미지 카테고리 없이도 계속 정상 표시되는가?
- 카드 내용이 고정 크기를 넘칠 때 텍스트를 잘라내는가, 폰트를 줄이는가? (콘텐츠 손실 최소화 방향으로 결정 필요)

## Requirements *(mandatory)*

### Functional Requirements

**이미지 라이브러리**

- **FR-001**: 시스템은 개인 투자자 카드 뉴스에 활용할 이미지 라이브러리(80-120장)를 구축하고 서비스 정적 파일(`/public`)로 제공해야 한다. 이미지는 WebP 포맷으로 통일하고 최대한 압축하여 총 용량을 최소화한다. 초기 이미지는 외부 이미지 서비스(Unsplash/Pexels)에서 수동 수집한다
- **FR-002**: 이미지 라이브러리는 아래 6가지 역할 분류를 모두 포함해야 한다:
  - **테마**: 주가 상승/하락, 환율, AI·반도체, 실적, 경고, 성장, 소비 등 투자 맥락 테마 (최소 8종)
  - **감정**: 불안·공포, 기대·FOMO, 결정의 순간, 유머·공감 등 개인 투자자 심리 (최소 4종)
  - **환경**: 한국 증권거래소, 미국 증권거래소, 금융가·도심, 거래 화면 (최소 4종)
  - **기업 실사**: 주요 기업(한국·미국 상위 10-20개사) 대표 이미지, 건물·공장, 로고·브랜드 (주요사 개별 + 일반 기업 대체 이미지)
  - **상징**: 화폐·금·코인, 차트 그래픽, 인포그래픽 요소 (최소 4종)
  - **행동**: 스마트폰 투자 화면, 투자자 분석 장면 (최소 2종)
- **FR-003**: 각 이미지 키에는 역할 분류·설명이 메타데이터로 관리되어, AI와 관리자가 키의 용도를 파악할 수 있어야 한다
- **FR-004**: 이미지 라이브러리는 초기 구축 이후 키를 추가할 수 있는 구조여야 한다

**이미지 키 시스템**

- **FR-005**: 시스템은 카드 시각 요소에 이미지 라이브러리의 키를 참조하는 `imgCategory` 필드를 지원해야 한다
- **FR-006**: AI 파이프라인은 카드 생성 시 자유형 텍스트 대신 라이브러리에 정의된 키 목록에서 가장 적합한 키를 선택해야 한다. AI에게는 키 목록과 각 키의 용도 설명이 제공된다
- **FR-007**: AI가 라이브러리에 없는 키를 생성할 경우, 시스템은 자동으로 기본 키로 폴백하여 렌더링 오류 없이 카드를 표시해야 한다
- **FR-008**: 기존 `CardVisual` gradient 필드는 하위 호환을 위해 유지되어야 하며, `imgCategory`와 공존해야 한다
- **FR-009**: `imgCategory`가 없는 기존 카드는 기존 gradient 방식으로 계속 표시되어야 한다

**일관된 시각 언어**

- **FR-010**: 이미지 라이브러리 전체에서 시각 톤을 일관되게 유지해야 한다: 상승=초록 계열, 하락=빨강 계열, 중립=회색/네이비 계열의 색상 가이드라인 준수
- **FR-011**: 감정 이미지는 금융 콘텐츠의 신뢰도를 훼손하지 않는 범위에서 선별되어야 한다 (공포 과잉·자극적 표현 배제)

**고정 카드 크기**

- **FR-012**: 카드 뉴스의 모든 카드는 **4:5 종횡비**로 고정되어야 한다. 카드 타입이나 콘텐츠 양에 관계없이 카드 외부 크기는 변하지 않는다
- **FR-013**: 카드 내용이 고정 크기를 초과하는 경우, 카드 외부 크기를 늘리지 않고 내부에서 처리(말줄임, 폰트 축소, 내부 스크롤 등)해야 한다
- **FR-014**: 고정 크기는 반응형으로 동작해야 한다. 화면 크기가 달라져도 모든 카드의 종횡비는 동일하게 유지된다

### Key Entities

- **이미지 라이브러리 (Image Library)**: 역할 분류별로 구조화된 이미지 자산 집합. 각 이미지는 고유 키, 역할 분류, 설명, 자체 호스팅 URL을 가짐
- **이미지 키 (Image Key)**: AI가 카드 생성 시 선택하는 식별자. 라이브러리의 특정 이미지 또는 이미지 그룹을 참조함
- **CardVisual (확장)**: 기존 gradient 필드에 `imgCategory` 필드가 추가된 형태. 두 필드 중 하나만 있거나 둘 다 있을 수 있음
- **기본 키 (Fallback Key)**: 라이브러리에 없는 키 또는 누락 키 발생 시 사용되는 안전망 이미지 키

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 이미지 라이브러리 6개 역할 분류(테마·감정·환경·기업·상징·행동) 전체에 이미지가 구비되어, AI가 다양한 뉴스 유형에 대해 카드별로 다른 이미지 키를 선택할 수 있다
- **SC-002**: AI 파이프라인이 생성하는 이미지 키 중 라이브러리 외 키(오류 키) 비율이 2% 이하이다
- **SC-003**: 기존 gradient 기반 카드는 이 기능 도입 후에도 깨짐 없이 동일하게 표시된다
- **SC-004**: 라이브러리 외 키가 입력되더라도 카드 렌더링 오류 발생 횟수가 0이다 (폴백으로 항상 표시됨)
- **SC-005**: 주요 기업(한국·미국 합산 10개 이상) 관련 카드에서 해당 기업을 연상할 수 있는 이미지가 표시된다
- **SC-006**: 카드 타입이 다른 카드들을 연속으로 탐색할 때 카드 간 높이 변화로 인한 스크롤 점프가 발생하지 않는다
- **SC-007**: 콘텐츠 양이 많은 카드와 적은 카드가 섞인 이슈에서도 모든 카드가 동일한 크기로 표시된다

## Assumptions

- 이미지는 WebP 포맷으로 통일한다. 원본(JPEG/PNG)을 WebP quality ~80으로 변환하여 장당 50-150KB, 전체 20MB 이하를 목표로 한다
- 이미지 파일은 `/public/images/cards/` 하위에 역할 분류별 폴더로 관리하며 git 레포에 포함한다
- S3/CDN으로의 이전은 이번 스코프에서 제외한다. 트래픽·운영 비용이 문제가 되는 시점에 별도 이슈로 분리한다
- 카드 종횡비는 **4:5**로 고정한다. 텍스트+이미지 조합의 인포그래픽 형식에 최적이며, 소셜 공유 시에도 별도 변환이 불필요하다
- 기업 전용 이미지는 저작권 문제가 없는 범위(공장 외관, 건물, 제품 사진 등)에서 수집한다. 로고 이미지는 상표권 이슈가 있으므로 건물·제품 사진으로 대체한다
- 초기 라이브러리 규모는 약 80-120장으로 시작하며, 이후 점진적으로 확대한다
- 각 이미지 키에 대응하는 이미지가 여러 장인 경우 이번 스코프에서는 고정 대표 이미지 1장을 사용한다 (랜덤 선택은 추후 확장)
- `imgCategory`는 신규 생성 카드에만 적용하며, 기존 DB 카드는 마이그레이션하지 않는다
- 시각 톤 가이드(상승=초록, 하락=빨강 등)는 이미지 수집 기준으로 활용한다

## Out of Scope

- S3/CDN 이전 및 외부 이미지 호스팅 (별도 이슈로 분리)
- 카드별 개별 종횡비 커스터마이징
- 이미지 랜덤 선택 및 A/B 테스트
- 기존 DB 카드의 이미지 소급 적용

## Dependencies

- 이슈 #99 (092-card-type-redesign): `CardVisual` 타입 및 기존 카드 구조가 이 기능의 확장 대상임
- 이미지 수집 작업: 라이브러리 구축은 구현 전 또는 병행하여 진행해야 하는 선행 작업
