**Source**: Claude 단일 대화 실험 (https://claude.ai/share/3b0e1e30-f8f2-4f47-8ad0-b2ccf1f17e83)

**Purpose**: 파이프라인 system prompt 개선을 위한 설계 인사이트 정리

## 1. 사례 개요

사용자가 Claude와의 단일 대화에서 고품질 카드 뉴스를 생성한 경험을 분석한다. 대화에서 Claude는 별도의 프레임워크 없이 자체 디자인 판단(Design Skills)을 활용해 다음 흐름으로 카드를 구성했다.

### 1.1 전체 프로세스 흐름

```
[1단계] 종목 선정
어떤 종목이 확인할 만한지 파악
→ 당일 거래량·등락 상위, 시장 화제 종목 기준

[2단계] 뉴스 리서치
해당 종목의 주요 이슈와 가격 변동 이유 파악
→ 기사 제목·본문에서 핵심 모멘텀 추출

[3단계] 커뮤니티 반응 수집
투자자 커뮤니티(카페·갤러리 등)의 실제 반응 파악
→ 감정 어조(기대/우려/관망) 분류

[4단계] 카드 구성
"왜 올랐나?" "더 오를까?" 궁금해할 독자를 위해 편집
→ 투자자 관점의 스토리라인으로 카드 배열

[5단계] 디자인 적용
Claude 자체 Design Skills 활용
→ 종목 분위기에 맞는 색상 팔레트 선택
```

## 2. 내러티브 아키텍처

### 2.1 투자자 질문 중심 스토리라인

효과적인 카드 뉴스는 독자의 질문에 순서대로 답변하는 구조를 갖는다:

| 순서 | 독자 질문 | 카드 타입 | 역할 |
|------|-----------|-----------|------|
| 1 | "뭔 일이야?" | `cover` | 결론과 핵심 수치 제시 |
| 2 | "왜 올랐어?" | `reason` | 주요 모멘텀 설명 |
| 3 | "더 오를 수 있나?" | `bullish` | 상승 근거 |
| 4 | "리스크는?" | `bearish` | 하락 근거/주의사항 |
| 5 | "다들 어떻게 봐?" | `community` | 시장 반응 |
| 6 | "얼마나 올랐나 수치로?" | `stats` | 정량 데이터 |
| 7 | "어디서 본 건데?" | `source` | 출처 신뢰성 |

### 2.2 Cover 카드 설계 원칙

Cover 카드는 30자 내외의 핵심 메시지로 독자를 잡아야 한다.

**좋은 예시**:
```
title: "삼성전자\n6개월 만에 7% 급등"
sub: "+6.93% · 2025-01-15"
```

**나쁜 예시**:
```
title: "삼성전자 주가 상승"  ← 정보 없음
sub: "오늘 상승"             ← 수치 없음
```

**원칙**:
- title 첫 줄: 종목명 또는 핵심 결론
- title 둘째 줄(있다면): 수치 또는 핵심 이벤트 (`\n` 분리)
- sub: "±N% · YYYY-MM-DD" 형식

### 2.3 카드 생략 기준

근거 없는 카드는 만들지 않는다:

- `bullish` 카드: 긍정 전망 근거 기사 없으면 생략
- `bearish` 카드: 리스크 또는 하락 근거 없으면 생략
- `community` 카드: 커뮤니티 반응을 유추할 기사 없으면 생략
- `stats` 카드: 의미 있는 수치를 추출할 수 없으면 생략

**최소 구성**: cover → reason 또는 bullish → source (3장)

## 3. 비주얼 팔레트 설계

### 3.1 Claude의 자체 Design Skills 활용 패턴

Claude는 단일 대화에서 종목 분위기에 맞게 색상을 선택했다. 이 패턴을 system prompt에 명시적으로 안내함으로써 파이프라인에서도 동일 품질을 재현한다.

### 3.2 분위기별 팔레트 가이드

#### 🔴 상승 · 강세 (Bullish Momentum)

```
분위기: 활기, 에너지, 기대감
bg_from → bg_via → bg_to: 어두운 warm → 중간 → 밝은 warm
accent: vivid 대비색

예시 팔레트 A (레드-오렌지):
  bg_from: #1a0505   bg_via: #3d0f0f   bg_to: #6b1a1a
  accent: #ff6b35

예시 팔레트 B (골드-앰버):
  bg_from: #0d0900   bg_via: #2a1f00   bg_to: #4a3500
  accent: #f5a623

예시 팔레트 C (그린-emerald, 코스피 상승):
  bg_from: #001a0d   bg_via: #003320   bg_to: #005533
  accent: #00c853
```

#### 🔵 하락 · 약세 · 리스크 (Bearish / Risk)

```
분위기: 우려, 침착, 경계감
bg_from → bg_via → bg_to: 어두운 cool → 중간 cool → 조금 밝은 cool
accent: cold 대비색

예시 팔레트 A (네이비-인디고):
  bg_from: #050514   bg_via: #0d0d2b   bg_to: #1a1a4a
  accent: #4fc3f7

예시 팔레트 B (슬레이트-블루):
  bg_from: #080c12   bg_via: #111827   bg_to: #1e2d40
  accent: #60a5fa

예시 팔레트 C (다크 퍼플-위기감):
  bg_from: #0c0514   bg_via: #1a0a2e   bg_to: #2d1050
  accent: #c084fc
```

#### ⚪ 중립 · 복합 (Neutral / Mixed)

```
분위기: 차분, 분석적, 전문적
bg_from → bg_via → bg_to: 모노크롬 다크

예시 팔레트 A (차콜-그레이):
  bg_from: #0a0a0a   bg_via: #1a1a1a   bg_to: #2d2d2d
  accent: #e2e8f0

예시 팔레트 B (다크 청록-중립):
  bg_from: #040d0d   bg_via: #0a1f1f   bg_to: #143333
  accent: #5eead4
```

### 3.3 팔레트 일관성 규칙

동일 이슈의 모든 카드는 같은 팔레트 계열을 사용한다:

```
✅ 올바른 예: 모든 카드가 레드-오렌지 계열
  cover:     bg_from #1a0505 → accent #ff6b35
  reason:    bg_from #1c0606 → accent #ff7f50
  bullish:   bg_from #180404 → accent #ff5722

❌ 잘못된 예: 카드마다 다른 계열
  cover:     bg_from #1a0505 (레드)
  reason:    bg_from #050514 (네이비) ← 계열 불일치
  bullish:   bg_from #0a0a0a (모노크롬) ← 계열 불일치
```

## 4. 카드 타입별 콘텐츠 가이드

### 4.1 reason 카드

**목적**: "왜 올랐나/내렸나?"에 대한 핵심 이유 제시

```
tag: '배경' | '모멘텀' | '이슈'
title: 이유를 명사구로 (예: "반도체 업황 개선 기대")
body: 2~3문장, 사실 위주
stat (선택): 가장 임팩트 있는 단일 수치
  예: { label: "외인 순매수", value: "2,847억" }
sources: 관련 기사 1~3개
```

### 4.2 bullish 카드

**목적**: 상승 또는 추가 상승 가능성의 근거

```
tag: '상승 논거' | '매수 근거' | '호재'
title: 긍정 전망을 명사구로
body: 왜 더 오를 수 있는지 근거 중심
stat (선택): 목표주가, 컨센서스, 성장률 등
sources: 뒷받침 기사 1~3개
```

### 4.3 bearish 카드

**목적**: 하락 또는 리스크 요인 제시 (균형적 시각)

```
tag: '리스크' | '하락 논거' | '주의'
title: 위험 요인을 명사구로
body: 구체적 리스크 내용
stat (선택): 손익분기 수치, 하락 시나리오 등
sources: 관련 기사 1~3개
```

### 4.4 community 카드

**목적**: 투자자 커뮤니티의 실제 반응 연상

**어조 가이드**: 실제 개인 투자자 커뮤니티(주식 갤러리, 네이버 종목 토론, 카카오 오픈채팅 등)의 대화 스타일을 반영. 공식적이지 않아도 되며, 감정이 살아있어야 한다.

```
quotes 예시:

positive mood:
  "이거 진짜 간다. 외인이 이렇게 사모으는 거 처음 봤음"
  "실적 발표 때 터지겠는데. 잘 잡았다"

negative mood:
  "고점에서 또 물렸노... 이번엔 진짜 손절 각임"
  "뭔 악재가 더 있는 거 아냐? 너무 갑자기 떨어지네"

neutral mood:
  "좀 더 지켜봐야 할 것 같은데, 아직 확신 없음"
  "오를 수도 내릴 수도 있는 구간 같음"
```

**생략 기준**: 기사에서 커뮤니티 반응을 유추할 수 없으면 community 카드 생략. 억지로 만들면 오히려 신뢰를 해친다.

### 4.5 stats 카드

**목적**: 투자 판단에 실제로 필요한 정량 정보 제공

**이슈 유형별 우선 지표**:

| 이슈 유형 | 우선 지표 | 예시 |
|-----------|-----------|------|
| 주식 (상승) | 등락률, 거래량, 외인순매수, 52주 고/저 | "거래량: 1,234만주 (평균 대비 3.2배)" |
| 주식 (하락) | 등락률, 거래량, 기관순매도, 손절대 | "기관 순매도: -1,847억" |
| 주식 (실적) | EPS, PER, 매출성장률, 영업이익 | "영업이익: 2.3조 (전년비 +34%)" |
| 지수 (KOSPI/나스닥) | 등락, 거래대금, 외인비중 | "거래대금: 12.4조 (최근 1달 최고)" |
| 통화 (USD-KRW) | 환율, 변동폭, 달러인덱스 | "달러인덱스: 104.3 (+0.4%)" |

**나쁜 examples (피해야 할 items)**:
```
❌ { label: "날짜", value: "2025-01-15" }       ← 타임스탬프 반복
❌ { label: "종목명", value: "삼성전자" }        ← 이미 cover에 있음
❌ { label: "상태", value: "상승" }              ← 정량 정보 없음
```

## 5. system prompt 구조 개선 방향

기존 `buildSystemPrompt()`의 구조:
```
[역할] → [카드 타입 카탈로그] → [제약 규칙] → [콘텐츠 규칙]
```

개선 후 추가할 섹션:
```
[역할] → [카드 타입 카탈로그] → [내러티브 흐름 가이드] (신규)
       → [비주얼 팔레트 가이드] (신규)
       → [카드별 콘텐츠 가이드] (신규, 기존 카탈로그 보강)
       → [제약 규칙] → [콘텐츠 규칙]
```

### 5.1 내러티브 흐름 섹션 (신규)

```markdown
## 내러티브 흐름

투자자는 다음 순서로 궁금해한다:
1. "뭔 일이야?" → cover: 핵심 결론 + 수치
2. "왜 이런 거야?" → reason: 변동 이유
3. "더 오를까?" → bullish: 상승 근거
4. "리스크는 없어?" → bearish: 하락 근거
5. "다들 어떻게 봐?" → community: 시장 반응
6. "수치로 확인해줘" → stats: 정량 데이터
7. "어디서 봤어?" → source: 출처

이 순서를 따라 카드를 배열한다. 근거 없는 카드는 생략한다.
```

### 5.2 비주얼 팔레트 섹션 (신규)

이슈의 전반적 분위기를 파악하고 팔레트를 선택:

- **상승·강세 모멘텀**: warm 계열 (레드-오렌지-골드, hue 0–60) 또는 vivid green (hue 100–150)
- **하락·리스크**: cool 계열 (네이비-인디고-퍼플, hue 200–280)
- **중립·복합**: 모노크롬 다크 또는 slate

이슈 내 모든 카드는 동일 팔레트 계열 유지. bg_from은 가장 어둡고, bg_to로 갈수록 약간 밝아지는 그라디언트.

## 6. 현재 system prompt 대비 Gap 분석

| 항목 | 현재 (v1) | 목표 (v2) |
|------|-----------|-----------|
| 내러티브 순서 | 없음 (카드 타입 목록만) | 투자자 질문 흐름 명시 |
| cover title 가이드 | "핵심 요점을 \n으로 강조" | "투자자 질문/결론 + 수치" |
| 비주얼 팔레트 | hex 코드 형식만 | 분위기별 색상 계열 + 예시 |
| 팔레트 일관성 | 없음 | 이슈 단위 동일 계열 명시 |
| community 어조 | 없음 | 커뮤니티 스타일 명시 + 생략 기준 |
| stats 콘텐츠 | 없음 | 이슈 유형별 우선 지표 명시 |
| 카드 생략 기준 | "근거 약하면 생략" | 타입별 생략 조건 명시 |
