# Feature Specification: 워크트리 환경변수 관리 자동화

**Feature Branch**: `001-worktree-env-setup`
**Created**: 2026-03-05
**Status**: Draft

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 새 워크트리에서 환경변수 즉시 사용 (Priority: P1)

개발자가 새 git worktree를 생성한 직후, 단일 명령어 하나로 `.env.local`을 워크트리에 연결하여 앱을 즉시 실행할 수 있다. 메인 저장소의 원본 환경변수 파일을 복사하거나 재입력하는 과정이 없다.

**Why this priority**: 워크트리마다 매번 수동으로 환경변수를 복사하는 것이 현재 가장 큰 반복 작업이며, 이 스토리만 해결되어도 핵심 목표를 달성한다.

**Independent Test**: 새 워크트리를 생성한 후 setup 명령어를 실행하고, `npm run dev`가 환경변수 오류 없이 정상 구동되는지 확인.

**Acceptance Scenarios**:

1. **Given** 새 워크트리가 생성되어 있고 메인 저장소에 `.env.local`이 존재할 때, **When** 개발자가 setup 명령어를 실행하면, **Then** 워크트리에서 메인 저장소의 환경변수를 참조할 수 있어 앱이 정상 구동된다.
2. **Given** setup 명령어를 실행한 이후, **When** 메인 저장소의 `.env.local` 값이 변경되면, **Then** 워크트리에서도 변경된 값이 즉시 반영된다 (재실행 불필요).
3. **Given** setup 명령어를 실행하려 할 때 메인 저장소에 `.env.local`이 없으면, **Then** 명확한 오류 메시지로 원본 파일이 없음을 안내한다.

---

### User Story 2 - 워크트리 환경변수 설정 상태 확인 (Priority: P2)

개발자가 현재 워크트리의 환경변수 연결 상태를 빠르게 확인할 수 있다. 연결이 제대로 되어 있는지, 원본 파일이 유효한지 한눈에 알 수 있어 디버깅 시간을 줄인다.

**Why this priority**: P1이 없으면 의미 없지만, 문제 발생 시 원인 파악 속도를 높여 개발자 경험을 향상시킨다.

**Independent Test**: setup 완료 후 상태 확인 명령어를 실행하여 "연결됨" 또는 "연결 안 됨" 메시지가 출력되는지 확인.

**Acceptance Scenarios**:

1. **Given** 워크트리에 환경변수가 정상 연결되어 있을 때, **When** 상태 확인을 실행하면, **Then** 연결 상태와 원본 파일 경로가 표시된다.
2. **Given** 워크트리에 환경변수가 연결되지 않았을 때, **When** 상태 확인을 실행하면, **Then** 연결이 없음을 알리고 setup 명령어 실행 방법을 안내한다.

---

### User Story 3 - 워크트리 생성 문서에서 절차 확인 (Priority: P3)

신규 개발자 또는 절차를 잊은 개발자가 문서를 통해 워크트리 생성 후 해야 할 환경변수 설정 절차를 명확히 파악할 수 있다.

**Why this priority**: 스크립트만 있고 문서가 없으면 발견하기 어렵다. 온보딩과 장기적 유지보수를 위해 필요하지만, 스크립트 기능이 선행되어야 한다.

**Independent Test**: 문서를 읽은 개발자가 스크립트 없이 절차를 따라 수동으로도 환경변수를 설정할 수 있는지 확인.

**Acceptance Scenarios**:

1. **Given** 개발자가 새 워크트리를 생성한 직후, **When** 프로젝트 문서에서 환경변수 설정 절차를 검색하면, **Then** 명확한 단계별 가이드와 실행할 명령어를 찾을 수 있다.
2. **Given** 문서에 기술된 절차를 따를 때, **When** 각 단계를 순서대로 실행하면, **Then** 환경변수 관련 오류 없이 앱이 실행된다.

---

### Edge Cases

- 메인 저장소에 `.env.local`이 없는 상태에서 setup을 실행하면 어떻게 처리하는가?
- 워크트리에 이미 `.env.local`(파일 또는 연결)이 존재하는 상태에서 setup을 재실행하면 기존 것을 덮어쓰는가?
- `.env.local`에 워크트리별로 다른 값이 필요한 경우(예: 다른 DB 환경)에 어떻게 대응하는가?
- 워크트리를 삭제할 때 연결된 환경변수 연결이 메인 저장소에 영향을 주지 않는가?
- Windows 또는 비-Unix 환경에서 동일한 절차가 동작하는가?

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 개발자는 새 워크트리 생성 후 단일 명령어로 환경변수 설정을 완료할 수 있어야 한다.
- **FR-002**: 환경변수 setup은 메인 저장소의 원본 `.env.local`을 기준으로 동작해야 하며, 값을 재입력할 필요가 없어야 한다.
- **FR-003**: 메인 저장소의 `.env.local`이 변경될 경우, 연결된 워크트리에서도 추가 작업 없이 변경된 값이 반영되어야 한다.
- **FR-004**: setup 실행 결과 `.env.local` 또는 비밀값이 git에 커밋되는 일이 없어야 한다.
- **FR-005**: 메인 저장소의 `.env.local`이 존재하지 않을 경우, setup은 실패하고 사용자에게 명확한 안내 메시지를 제공해야 한다.
- **FR-006**: 워크트리에 이미 환경변수 연결이 존재하는 경우, setup 재실행 시 사용자에게 확인을 요청하거나 덮어쓰기 여부를 명시해야 한다.
- **FR-007**: 개발자는 현재 워크트리의 환경변수 연결 상태를 확인할 수 있어야 한다.
- **FR-008**: 워크트리 환경변수 설정 절차가 프로젝트 문서에 명시되어야 한다.

### Key Entities

- **메인 환경변수 파일**: 메인 저장소 루트에 위치한 `.env.local`. 모든 워크트리가 공유하는 단일 원본.
- **워크트리 환경변수 참조**: 각 워크트리 루트에 생성되는 환경변수 접근 수단. 원본을 직접 참조하거나 복사한 형태.
- **Setup 도구**: 워크트리에서 실행하여 환경변수 참조를 생성하는 스크립트 또는 명령어.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 새 워크트리 환경변수 setup이 명령어 실행 후 10초 이내에 완료된다.
- **SC-002**: 환경변수 setup을 위한 수동 복사 또는 값 재입력 단계가 0회여야 한다.
- **SC-003**: 메인 저장소 `.env.local` 변경 후 워크트리에서 별도 작업 없이 즉시 반영된다 (추가 명령어 0회).
- **SC-004**: setup 후 `npm run dev` 실행 시 환경변수 관련 오류가 발생하지 않는다.
- **SC-005**: `.env.local` 또는 시크릿 패턴을 포함한 파일이 git 커밋에 포함되는 경우가 0건이어야 한다.

## Assumptions

- 모든 워크트리는 동일한 환경변수 세트를 사용한다고 가정한다 (워크트리별 별도 환경이 필요한 경우는 Out of Scope).
- 메인 저장소 루트의 `.env.local`이 항상 최신 원본 역할을 한다.
- 개발자는 macOS 또는 Linux 환경을 기본으로 사용한다 (Windows 지원은 Best effort).
- `.env.local`은 이미 `.gitignore`에 등록되어 있어 실수로 커밋되는 것을 방지한다.
