# Feature Specification: 파이프라인 토큰 비용 최적화

**Feature Branch**: `054-pipeline-cost-opt`
**Created**: 2026-03-24
**Status**: Draft
**GitHub Issue**: #85

## User Scenarios & Testing *(mandatory)*

### User Story 1 — 파이프라인이 관련 기사만 선별해 AI에 전달한다 (Priority: P1)

운영자가 수동 또는 cron으로 파이프라인을 실행하면, 수집된 기사 전체(오늘 123건) 대신 투자 관련 기사 10건으로 압축된 뒤 카드가 생성된다.

**Why this priority**: 1회 ~$0.23 → ~$0.05 비용 절감의 핵심이며, 관련 없는 기사가 줄어 카드 품질도 함께 개선된다.

**Independent Test**: 파이프라인 실행 후 로그에서 AI에 전달된 기사 수가 10건 이하이고 이슈가 정상 생성되면 성공.

**Acceptance Scenarios**:

1. **Given** 소스에서 기사 123건이 수집됐을 때, **When** 파이프라인이 AI 카드 생성 단계에 진입하면, **Then** AI에 전달되는 기사는 최대 10건이다.
2. **Given** 소스당 기사가 30건을 초과할 때, **When** 1차 게이트가 실행되면, **Then** 해당 소스에서 최신 30건만 후속 처리에 사용된다.
3. **Given** 투자와 무관한 기사(날씨, 스포츠 등)가 섞여 있을 때, **When** 필터링이 완료되면, **Then** 결과 10건은 투자 관련 기사 위주로 구성된다.
4. **Given** 동일 종목 기사가 여러 건 있을 때, **When** 필터링이 완료되면, **Then** 대표 기사 1건만 포함되어 중복이 제거된다.
5. **Given** Haiku 필터링 단계에서 오류가 발생했을 때, **When** 파이프라인이 계속 실행되면, **Then** 필터를 건너뛰고 소스별 30건 상한이 적용된 원본 기사로 Sonnet을 호출한다.

---

### User Story 2 — 운영자가 실행별 비용을 Admin에서 확인한다 (Priority: P1)

Admin 파이프라인 탭에서 각 실행 로그 행에 토큰 사용량과 추산 비용(USD)이 표시된다.

**Why this priority**: 비용 절감 효과를 수치로 검증할 수 없으면 최적화 방향을 판단하기 어렵다.

**Independent Test**: 파이프라인 실행 후 Admin에서 해당 행에 비용 수치가 표시되면 독립 검증 가능.

**Acceptance Scenarios**:

1. **Given** 파이프라인이 정상 완료됐을 때, **When** Admin 파이프라인 로그 목록을 보면, **Then** 해당 행에 추산 비용(USD)이 표시된다.
2. **Given** 파이프라인이 기사 수집 후 AI 단계에서 실패했을 때, **When** 로그를 보면, **Then** 비용은 $0.00 또는 미집계로 표시된다.
3. **Given** 여러 날치 로그가 있을 때, **When** 목록을 확인하면, **Then** 각 행마다 비용이 일관되게 표시된다.

---

### User Story 3 — 파이프라인이 최대 3개 이슈만 생성한다 (Priority: P2)

개발 단계에서 AI 출력 토큰을 제한하기 위해 이슈 수를 3개로 제한한다.

**Why this priority**: 출력 토큰 감소로 비용과 응답 잘림(truncation) 안정성이 함께 개선된다.

**Independent Test**: 기사 10건으로 파이프라인 실행 후 생성된 이슈가 최대 3개이면 성공.

**Acceptance Scenarios**:

1. **Given** 기사 10건이 AI에 전달됐을 때, **When** 이슈 생성이 완료되면, **Then** 생성된 이슈는 3개 이하이다.
2. **Given** 기사 내용이 다양해 5개 이슈 생성이 가능한 경우에도, **When** 이슈 생성이 완료되면, **Then** 상위 3개만 반환된다.

---

### Edge Cases

- 소스가 1개이고 기사가 5건뿐일 때: 30건 상한 미적용, 5건 전체가 필터 입력.
- 필터링 후 기사가 0건일 때: 이슈 생성 없이 파이프라인 종료, 로그에 기록.
- API 응답에 usage 정보가 없는 경우: 비용 필드를 null로 저장하고 Admin에서는 "—"으로 표시.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 소스별로 최신 기사 최대 30건만 필터 단계에 전달해야 한다.
- **FR-002**: 시스템은 저비용 AI 모델을 사용해 전달받은 기사 중 투자 관련성이 높고 주제 중복이 없는 상위 10건을 선별해야 한다.
- **FR-003**: 카드 생성 AI에 전달되는 기사의 본문은 최대 500자로 제한해야 한다.
- **FR-004**: 시스템은 카드 생성 AI 응답에서 최대 3개 이슈만 생성해야 한다.
- **FR-005**: 필터링 단계에서 오류가 발생하면 필터를 건너뛰고 소스별 30건 상한 적용 기사 원본으로 카드 생성을 계속해야 한다.
- **FR-006**: 파이프라인 실행 로그에 각 AI 호출의 토큰 사용량(입력/출력)과 추산 비용(USD)을 저장해야 한다.
- **FR-007**: Admin 파이프라인 로그 목록에 실행별 추산 비용 컬럼을 표시해야 한다.

### Key Entities

- **FilterResult**: 필터링 결과 — 선별된 기사 목록, 사용 토큰, 오류 여부
- **TokenUsage**: 단계별 토큰 사용량 — 입력 토큰, 출력 토큰, 추산 비용(USD)
- **PipelineLog**: 기존 엔티티에 `tokens_input`, `tokens_output`, `estimated_cost_usd` 필드 추가

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 파이프라인 1회 실행 비용이 기존 대비 75% 이상 감소한다 (~$0.23 → ~$0.05 이하).
- **SC-002**: 카드 생성 AI에 전달되는 기사 수가 10건 이하로 제한된다.
- **SC-003**: 생성되는 이슈 수가 3개 이하로 제한된다.
- **SC-004**: Admin에서 매 실행의 추산 비용을 USD로 확인할 수 있다.
- **SC-005**: 필터 오류 시 파이프라인이 중단되지 않고 fallback으로 계속 실행된다.

## Assumptions

- 비용 추산은 다음 단가를 하드코딩해 계산 (실시간 API 요금과 다를 수 있음)
  - Haiku 4.5: 입력 $0.80/MTok, 출력 $4.00/MTok
  - Sonnet 4.5: 입력 $3.00/MTok, 출력 $15.00/MTok
- 이슈 수 상한(3개)은 코드 상수로 관리 (환경변수 설정 불필요, 단순성 우선)
- Haiku 필터에는 기사 제목과 요약만 전달 (content는 비용 절감을 위해 제외)
