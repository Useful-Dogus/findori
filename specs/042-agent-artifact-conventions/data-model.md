# Data Model: 에이전트 산출물 작성 규칙 통합

**Feature**: 042-agent-artifact-conventions
**Phase**: 1 — Design
**Date**: 2026-03-03

---

이 피처는 소스 코드 변경이 없는 **순수 문서 피처**다. "데이터 모델"은 공통 정책 문서의 구조(섹션 계층)를 정의한다.

---

## 공통 정책 문서 구조 (`docs/artifact-conventions.md`)

```
docs/artifact-conventions.md
│
├── # 에이전트 산출물 작성 규칙 (제목)
│
├── ## 1. 번호 정렬 체계
│   ├── GitHub 이슈 번호 → 브랜치 번호 → 스펙 디렉터리 번호
│   └── 3자리 제로패딩 규칙
│
├── ## 2. 스펙 (spec.md)
│   ├── 파일 경로 패턴
│   └── speckit 워크플로우 경로
│
├── ## 3. GitHub 이슈
│   ├── 제목 형식
│   └── 필수 섹션 목록 (배경, 범위 In/Out, 완료 기준)
│
├── ## 4. 커밋 메시지
│   ├── Tier 1: [Issue #N] 형식 (이슈 구현 주 커밋)
│   ├── Tier 2: Conventional Commits (보조/수정 커밋)
│   ├── 허용 타입 집합 표
│   └── hotfix 예외 규칙
│
├── ## 5. PR (Pull Request)
│   ├── 제목 형식
│   ├── 필수 섹션 (Summary, Test plan, Closes/Refs)
│   ├── 선택 섹션 (Changes)
│   └── 링크 키워드 규칙 (Closes / Part of / Refs)
│
├── ## 6. 코드 품질 기준
│   ├── 가독성
│   ├── 예측 가능성
│   ├── 응집도
│   └── 결합도 & 기본 성능
│
└── ## 7. hotfix 예외 흐름
    ├── hotfix 정의
    ├── 최소 필수 항목
    └── 생략 가능 항목
```

---

## 섹션 간 관계

```
번호 정렬 체계
    ↓ 참조
커밋 메시지 (Tier 1: [Issue #N])
    ↓ 참조
PR 제목 형식 ([Issue #N] <description>)
    ↓ 포함
Closes #N / Refs #N 링크 키워드

hotfix 예외 흐름
    ↓ 오버라이드
커밋 메시지 (hotfix: 타입)
PR 필수 섹션 (최소화)
```

## 허용 커밋 타입 집합

| 타입       | 용도                                    | Tier |
|-----------|----------------------------------------|------|
| `feat`    | 새 기능 추가                             | 2    |
| `fix`     | 버그 수정                               | 2    |
| `hotfix`  | 긴급 프로덕션 버그 수정                   | 2    |
| `docs`    | 문서만 변경                              | 2    |
| `chore`   | 설정·도구·스크립트 변경 (프로덕션 무관)    | 2    |
| `refactor`| 동작 변경 없는 코드 구조 개선             | 2    |
| `test`    | 테스트 추가·수정                         | 2    |
| `style`   | 코드 포매팅·세미콜론 등 (로직 무관)       | 2    |
| `perf`    | 성능 개선                               | 2    |
| `ci`      | CI/CD 설정 변경                         | 2    |
| `[Issue #N]` | 이슈 구현 주 커밋                    | 1    |

> Tier 1(`[Issue #N]`)은 이슈 기반 구현의 주 커밋에만 사용. Tier 2(Conventional Commits)는 보조 커밋·hotfix·이슈 없는 수정에 사용.
