# Feature Specification: 공개 라우팅/SSR 진입 플로우

**Feature Branch**: `016-public-routing-ssr`
**Created**: 2026-03-19
**Status**: Draft
**GitHub Issue**: #16

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 홈 접근 시 오늘 피드로 자동 이동 (Priority: P1)

비로그인 사용자가 홈(/)에 접근하면, 가장 최근에 발행된 피드 날짜의 이슈 카드 스트림으로 즉시 이동한다.

**Why this priority**: 서비스의 첫 진입점이며 모든 공개 사용자가 거치는 필수 경로다. 오늘 날짜에 발행된 피드가 없는 경우 가장 최근 발행일로 이동하여 빈 화면 노출을 막는다.

**Independent Test**: 홈(/) 접근 → 리다이렉트 동작 확인만으로 독립 검증 가능하며, 카드 렌더러 없이도 날짜가 올바른 URL로 이동하는지 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 발행된 피드가 존재할 때, **When** 사용자가 `/`에 접근하면, **Then** 최신 발행 날짜(`/feed/YYYY-MM-DD`)로 즉시 리다이렉트된다.
2. **Given** 발행된 피드가 하나도 없을 때, **When** 사용자가 `/`에 접근하면, **Then** 빈 피드 안내 화면이 표시된다.
3. **Given** 피드 API 호출이 실패할 때, **When** 사용자가 `/`에 접근하면, **Then** 에러 안내와 새로고침 유도 UI가 표시된다.

---

### User Story 2 - 날짜별 피드 SSR 로드 (Priority: P1)

사용자가 `/feed/2026-03-19` 형태의 URL에 접근하면, 해당 날짜의 발행된 이슈 카드 스트림 데이터가 서버에서 사전 렌더링되어 즉시 표시된다.

**Why this priority**: 피드 소비의 핵심 화면으로, SSR로 초기 데이터를 미리 담아야 공유 링크 진입 시 즉각 열람이 가능하다.

**Independent Test**: 유효한 날짜 URL 접근 → HTTP 200 + 이슈 목록 포함 HTML 응답 확인. 카드 렌더러(#17) 없이도 이슈 메타데이터 노출로 독립 검증 가능하다.

**Acceptance Scenarios**:

1. **Given** 해당 날짜에 발행된 피드가 있을 때, **When** 사용자가 `/feed/[date]`에 접근하면, **Then** SSR로 이슈 목록 데이터가 포함된 페이지가 즉시 반환된다.
2. **Given** 해당 날짜에 발행된 피드가 없을 때, **When** 사용자가 `/feed/[date]`에 접근하면, **Then** 빈 날짜 안내 화면과 홈으로 이동하는 CTA가 표시된다.
3. **Given** 잘못된 날짜 형식(`/feed/abc`)으로 접근할 때, **When** 페이지가 로드되면, **Then** 404 페이지가 반환되고 홈으로 유도하는 링크가 제공된다.
4. **Given** 유효한 날짜지만 피드 API 호출이 실패할 때, **When** 페이지가 로드되면, **Then** 에러 안내와 새로고침 유도 UI가 표시된다.
5. **Given** 페이지 메타데이터가 필요할 때, **Then** `<title>` 및 OG 태그가 날짜 정보를 포함하여 설정된다.

### User Story 3 - 공유 링크 직접 진입 (이슈 단위) (Priority: P1)

사용자가 `/feed/2026-03-19/issue/[uuid]` 형태의 공유 링크를 받아 접근하면, 해당 이슈로 초점이 맞춰진 피드 화면이 SSR로 즉시 렌더링된다.

**Why this priority**: 비로그인 사용자의 공유 링크 진입은 서비스 확산의 핵심 경로다. 로그인 없이, 링크만으로 즉각 열람이 보장되어야 한다.

**Independent Test**: 유효한 이슈 공유 링크 접근 → HTTP 200 + 이슈 데이터 포함 HTML 응답 확인. 스와이프 UI(#18) 없이도 이슈 데이터가 SSR로 삽입되는지 독립 검증 가능하다.

**Acceptance Scenarios**:

1. **Given** 유효한 이슈 공유 링크일 때, **When** 비로그인 사용자가 접근하면, **Then** SSR로 해당 이슈 데이터가 포함된 페이지가 즉시 반환되고, 해당 이슈가 초기 선택 상태로 페이지에 전달된다.
2. **Given** 존재하지 않는 이슈 ID일 때, **When** 사용자가 접근하면, **Then** 404 페이지가 반환되고 홈으로 유도하는 링크가 제공된다.
3. **Given** 이슈 API 호출이 실패할 때, **When** 페이지가 로드되면, **Then** 에러 안내와 재시도 유도 UI가 표시된다.
4. **Given** 이슈 공유 링크 진입 시, **Then** OG 메타태그에 해당 이슈 제목이 설정되어 SNS 미리보기에 의미 있는 내용이 표시된다.

---

### Edge Cases

- 피드는 존재하나 승인된 이슈가 0건인 날짜 접근 시: 빈 피드 안내 화면 표시
- 미래 날짜 접근 시: 발행된 피드 없음으로 처리(빈 상태)
- `YYYY-MM-DD` 형식이지만 실존하지 않는 날짜(예: `2026-02-30`) 접근 시: 404 처리
- `/feed/latest` 경로 접근 시: 최신 발행 날짜로 리다이렉트

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: `/` 접근 시 서버에서 최신 발행 피드 날짜를 조회하여 `/feed/[date]`로 리다이렉트해야 한다.
- **FR-002**: 발행된 피드가 없을 때 `/` 접근 시 빈 피드 안내 화면을 노출해야 한다.
- **FR-003**: `/feed/[date]` 페이지는 SSR로 해당 날짜 피드 데이터(이슈 목록 + cards_data)를 사전 로드해야 한다.
- **FR-004**: `/feed/[date]/issue/[id]` 페이지는 SSR로 해당 이슈 데이터를 사전 로드하고, 해당 이슈가 초기 선택된 상태를 페이지 초기값으로 전달해야 한다.
- **FR-005**: 유효하지 않은 날짜 형식 또는 존재하지 않는 이슈 접근 시 404를 반환하고 홈 유도 CTA를 제공해야 한다.
- **FR-006**: 각 공개 라우트는 `<title>` 및 OG 메타태그(`og:title`, `og:description`, `og:url`)를 동적으로 설정해야 한다.
- **FR-007**: 피드/이슈 로드 실패(API 오류 포함) 시 에러 상태 화면을 표시하고, 사용자가 재시도하거나 홈으로 이동하는 경로를 즉시 인지할 수 있어야 한다.
- **FR-008**: 모든 공개 라우트는 비로그인 상태에서 완전히 열람 가능해야 한다.
- **FR-009**: `/feed/latest` 경로는 최신 발행 날짜로 리다이렉트해야 한다.

### Key Entities

- **Feed**: 특정 날짜의 발행 단위. `date`(YYYY-MM-DD), `status`(published만 공개 노출) 보유
- **Issue**: 피드 내 개별 이슈 카드 스트림. `id`(UUID), `title`, `entity_name`, `cards_data`, `tags`, `order` 보유
- **SSR 초기 상태**: 서버에서 사전 로드된 피드/이슈 데이터로, 클라이언트에 전달되어 카드 렌더러(#17)와 스와이프 UI(#18)가 소비

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 공유 링크 진입 시 이슈 데이터를 포함한 서버 응답이 1초 이내에 반환된다.
- **SC-002**: 홈(/) 접근 시 최신 피드 날짜로의 리다이렉트가 2초 이내에 완료된다.
- **SC-003**: 발행된 피드가 없거나 잘못된 날짜/이슈 ID로 접근 시, 100% 케이스에서 사용자가 홈 또는 재시도 경로를 즉시 확인할 수 있다.
- **SC-004**: 공유 링크를 통해 접근한 비로그인 사용자가 별도 액션(로그인, 버튼 클릭 등) 없이 이슈 데이터가 포함된 화면을 확인할 수 있다.
- **SC-005**: 각 라우트의 OG 메타태그가 올바르게 설정되어 SNS 링크 미리보기에 의미 있는 제목이 표시된다.

## Assumptions

- 공개 피드 API(`/api/feeds/[date]`, `/api/feeds/latest`, `/api/issues/[id]`)는 이미 구현되어 있다(#15 완료).
- 카드 렌더러(#17), 스와이프 UI(#18)는 이 스펙 범위 밖이다. 이 스펙은 SSR 데이터 패칭과 라우트 진입 흐름만 담당하며, 카드 UI 렌더링은 후속 이슈에서 구현한다.
- 공유 링크(`/feed/[date]/issue/[id]`) 진입 시 초기 선택 이슈 정보를 페이지 초기 상태로 전달하는 것까지만 담당하고, 실제 스와이프/포지셔닝 UI는 #18에서 구현한다.
- OG 이미지(`/api/og/issue/[id]`)는 별도 이슈(#25)에서 구현한다. 이 스펙에서는 `og:image` 태그에 기본 정적 이미지(`/og-default.png`)를 사용한다.
- 투자 자문 아님 고지(#24)는 별도 이슈에서 구현한다.
