# Research: 유저 피드 화면 UX 폴리싱

## Decision 1: 빈 마켓 컨텍스트 슬롯은 숨긴다

- Decision: 컨텍스트 섹션은 데이터가 실제로 존재하는 항목만 렌더링한다.
- Rationale: "준비중"이나 "업데이트 지연" 상태가 카드보다 먼저 노출되면 피드 전체가 미완성처럼 보인다.
- Alternatives considered:
  - 준비중 상태 유지: 사용자는 기능 미완성으로 인식할 가능성이 높아 기각.
  - 빈 슬롯은 유지하되 레이블만 숨김: 자리만 차지하는 저가치 UI라 기각.

## Decision 2: 카드 외부 설명성 UI는 최소화한다

- Decision: 소개 문구, 카드 사용법 문구, 반복 메타 정보, 저가치 태그를 제거하거나 축소한다.
- Rationale: 피드형 소비 경험에서는 카드 밖 문구가 많을수록 카드 몰입이 떨어진다.
- Alternatives considered:
  - 설명 문구를 더 짧게 다듬기: 설명이 남아 있는 한 문제의 본질은 유지되어 기각.
  - 툴팁이나 도움말 아이콘으로 이동: 초기 진입부가 여전히 복잡해져 기각.

## Decision 3: 카드 가독성은 렌더링 레이어에서 우선 보정한다

- Decision: 단어 단위 줄바꿈, 말줄임 완화, 이미지 오버레이 강화, 초기 로딩 상태 제어를 프론트엔드에서 우선 해결한다.
- Rationale: 이번 범위는 파이프라인 수정 없이 shipped UI 품질을 개선하는 데 있다.
- Alternatives considered:
  - 생성 단계 카피 규칙부터 수정: 효과는 크지만 범위가 커지고 원인 분리가 어려워 후속 이슈로 분리.
  - 카드별 수동 예외 처리: 유지보수성이 낮아 기각.

## Decision 4: 데스크톱에서는 완전한 비례 확대 대신 읽기 밀도 완화를 적용한다

- Decision: 큰 화면에서 카드 내부 타이포와 간격을 키워 빈약한 인상을 줄이되, 모든 카드 타입의 완전한 비례 스케일 시스템은 도입하지 않는다.
- Rationale: 이번 범위에서 해결 가능한 품질 개선이며, 리스크가 통제 가능하다.
- Alternatives considered:
  - 아무 조정도 하지 않음: 사용자 피드백 핵심을 놓쳐 기각.
  - 완전 비례 확대 시스템 도입: 카드 타입 전반 재설계가 필요해 후속 이슈로 분리.

## Decision 5: 면책 문구는 제거하지 않고 하단으로 이동한다

- Decision: 공개 레이아웃에서는 면책 문구를 콘텐츠 하단에 배치한다.
- Rationale: 법적/신뢰 측면에서 문구는 유지하되, 콘텐츠보다 앞서는 현재 위계는 조정해야 한다.
- Alternatives considered:
  - 완전 제거: 규제/정책 리스크가 있어 기각.
  - 접기 UI로 축소: 구현 복잡도 대비 이득이 작아 기각.
