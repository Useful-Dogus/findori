# Feature Specification: 유저 피드 화면 UX 폴리싱

**Feature Branch**: `115-feed-ux-polish`  
**GitHub Issue**: #115  
**Created**: 2026-03-31  
**Status**: Draft  
**Input**: User description: "유저 피드 화면 UX 개선. 테스트 유저 피드백을 제품 관점에서 재구성하고, 명시되지 않은 같은 맥락의 문제까지 함께 정리."

## 배경 및 문제 정의

테스트 유저 피드백을 종합하면 현재 유저 피드는 "카드뉴스를 소비하는 공간"이 아니라 "카드뉴스를 설명하는 화면"처럼 느껴진다. 이 문제는 개별 문구 몇 개의 문제가 아니라, 제품이 무엇을 전면에 두고 무엇을 뒤로 밀어야 하는지에 대한 우선순위가 흔들린 결과다.

핵심 문제는 네 가지다.

1. **콘텐츠보다 크롬이 앞선다**: 면책 문구, 소개 문구, 사용법 문구, 중복된 메타 정보가 카드보다 먼저 시선을 가져간다.
2. **카드가 완성된 편집물처럼 보이지 않는다**: 텍스트와 이미지가 따로 뜨고, 줄바꿈과 말줄임이 어색하며, 이미지가 텍스트를 방해한다.
3. **정보 노출 기준이 느슨하다**: 준비되지 않은 마켓 슬롯, 의미가 약한 태그, 중복된 카드 수 표기처럼 "있어서 보여주는 정보"가 많다.
4. **피드의 정신모델이 불명확하다**: 사용자는 인스타그램형 카드 피드를 기대하는데, 현재 화면은 제목, 태그, 요약, 설명 문구가 겹쳐 카드 자체의 몰입을 깨뜨린다.

이번 스펙은 **파이프라인 수정 없이 프론트엔드와 렌더링 레이어에서 해결 가능한 UX 품질 문제**를 우선 다룬다. 따라서 데스크톱 카드 스케일 완화, 카드 외부 메타 정보 축소, 제목 위계 재조정처럼 화면에서 바로 개선 가능한 항목은 이번 범위에 포함한다. 반면 카드 수 가변화나 생성 단계의 카피 편집 규칙처럼 파이프라인 성격이 강한 항목은 후속 이슈로 분리한다.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 카드가 첫 화면의 주인공이 된다 (Priority: P1)

독자가 피드에 들어오면 설명문이나 준비중 상태가 아니라, 바로 읽을 수 있는 카드 콘텐츠를 본다. 화면 상단은 피드를 여는 최소한의 맥락만 제공하고, 가장 먼저 소비해야 하는 카드가 우선 노출된다.

**Why this priority**: 첫 화면의 정보 위계는 제품의 완성도를 결정한다. 카드가 주인공이어야 하는 서비스에서 설명문과 보조 정보가 상단을 차지하면 신뢰가 떨어진다.

**Independent Test**: 피드 첫 진입 시 스크롤 없이 보이는 영역을 확인해 카드 진입부가 먼저 보이는지, 면책 및 설명성 문구가 상단에 없는지 검수한다.

**Acceptance Scenarios**:

1. **Given** 사용자가 피드 화면에 처음 진입했을 때, **When** 첫 화면을 본다, **Then** 면책 문구가 최상단에 있지 않고 카드 소비를 시작할 수 있는 핵심 콘텐츠가 먼저 보인다.
2. **Given** 상단 헤더 영역을 본다, **When** 정보 구성을 확인한다, **Then** 카드와 직접 관련 없는 설명 문구는 제거되어 있다.
3. **Given** 마켓 컨텍스트 데이터가 비어 있는 날짜다, **When** 피드를 연다, **Then** 비어 있는 슬롯이나 "준비중" 상태는 화면에 보이지 않는다.
4. **Given** 모바일 화면에서 피드를 연다, **When** 헤더와 이동 버튼 배치를 본다, **Then** 버튼과 텍스트가 한쪽으로 치우치지 않고 정상적인 폭 안에서 정렬된다.

---

### User Story 2 - 카드는 완성된 편집물처럼 읽힌다 (Priority: P1)

독자는 각 카드를 텍스트와 이미지가 하나로 편집된 결과물처럼 받아들인다. 카드 안의 텍스트는 잘리지 않고 읽히며, 이미지가 있어도 정보 전달이 우선된다.

**Why this priority**: 카드형 콘텐츠에서 편집 완성도는 곧 신뢰도다. 줄바꿈, 말줄임, 이미지 가독성, 로딩 어긋남은 모두 "자동 생성물"처럼 느끼게 만든다.

**Independent Test**: 다양한 카드 타입과 이미지가 있는 카드를 열어 이미지와 텍스트가 함께 나타나는지, 말줄임 없이 읽히는지, 단어 중간 줄바꿈이 없는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 카드가 처음 표시될 때, **When** 카드가 화면에 나타난다, **Then** 이미지와 텍스트가 따로 노출되지 않고 하나의 완성된 카드로 인식된다.
2. **Given** 카드에 한국어 본문과 출처명이 포함되어 있을 때, **When** 줄바꿈이 발생한다, **Then** 단어 중간이 부자연스럽게 끊기지 않는다.
3. **Given** 텍스트 양이 많은 카드다, **When** 카드를 읽는다, **Then** 핵심 텍스트가 말줄임 없이 끝까지 확인 가능하다.
4. **Given** 밝거나 대비가 강한 배경 이미지를 가진 카드다, **When** 카드를 읽는다, **Then** 텍스트는 이미지보다 우선적으로 읽힌다.

---

### User Story 3 - 피드 크롬은 카드 소비를 방해하지 않는다 (Priority: P2)

독자는 카드 중심으로 피드를 훑고, 필요한 최소 정보만 접한다. 카드 상하단의 중복 메타 정보, 의미가 약한 태그, 과한 사용법 안내 없이도 자연스럽게 카드를 넘길 수 있다.

**Why this priority**: 피드형 제품은 "카드 외부 정보"가 적을수록 집중도가 올라간다. 카드 바깥 설명이 많으면 피드가 아니라 문서처럼 느껴진다.

**Independent Test**: 이슈 카드 섹션을 열어 카드 외부에 반복적으로 노출되는 정보가 줄었는지, 별도 안내문 없이 카드 탐색이 가능한지 확인한다.

**Acceptance Scenarios**:

1. **Given** 사용자가 이슈 카드 섹션을 본다, **When** 카드 주변 정보를 확인한다, **Then** 카드 수, 제목, 키워드 같은 메타 정보는 중복되지 않고 필요한 위치에 한 번만 보인다.
2. **Given** 이슈 카드에 분류 태그가 표시된다, **When** 태그의 정보 가치를 평가한다, **Then** 카드 이해에 직접 기여하지 않는 태그는 기본 노출되지 않거나 최소화된다.
3. **Given** 사용자가 처음으로 카드 영역을 본다, **When** 카드를 넘긴다, **Then** 별도의 사용법 문구 없이도 다음 카드 이동 방법을 인지할 수 있다.
4. **Given** 사용자가 이슈 블록을 본다, **When** 제목과 메타 정보의 위계를 확인한다, **Then** 카드 자체가 주인공으로 보이도록 제목은 과도하게 강조되지 않고 캡션에 가까운 보조 정보처럼 정돈된다.

---

### User Story 4 - 큰 화면에서도 카드가 빈약해 보이지 않는다 (Priority: P2)

데스크톱 사용자는 큰 화면에서 카드 배경만 커진 빈 상자를 보는 대신, 카드 내부 정보도 화면 크기에 맞춰 더 여유롭고 선명하게 읽는다. 완전한 비례 확대 시스템은 아니어도, 최소한 큰 화면일수록 카드 내부 타이포와 간격이 빈약해 보이지 않아야 한다.

**Why this priority**: 이 문제는 사용자에게 즉각적으로 보이는 품질 문제이고, 파이프라인이 아니라 렌더링 레이어에서 완화할 수 있다. 이번에 포함해도 범위가 과도하게 번지지 않는다.

**Independent Test**: 대표 데스크톱 뷰포트에서 카드를 확인해 배경만 커진 느낌이 줄고, 본문과 수치 정보의 가독성이 개선되었는지 비교 검수한다.

**Acceptance Scenarios**:

1. **Given** 넓은 데스크톱 화면에서 카드를 본다, **When** 카드 크기와 내부 콘텐츠를 함께 확인한다, **Then** 카드 배경만 커진 느낌이 아니라 내부 텍스트와 핵심 정보도 더 읽기 좋은 크기와 간격으로 표시된다.
2. **Given** 카드 타입별로 수치, 본문, 출처, 버튼이 함께 표시된다, **When** 데스크톱 화면에서 카드를 읽는다, **Then** 좁은 모바일 기준 밀도를 그대로 확대한 답답한 레이아웃처럼 보이지 않는다.

### Edge Cases

- 마켓 데이터가 일부만 있는 경우에도 빈 슬롯 없이 데이터가 있는 항목만 보여야 한다.
- 이미지가 없거나 늦게 도착하는 카드도 미완성 화면처럼 보이지 않아야 한다.
- 긴 고유명사, 언론사명, 숫자 조합이 포함된 문장도 부자연스럽게 끊기지 않아야 한다.
- 태그를 줄인 뒤에도 사용자가 이슈의 주제 맥락을 잃지 않아야 한다.
- 제목 위계를 낮춘 뒤에도 사용자가 현재 읽는 이슈를 식별할 수 있어야 한다.
- 데스크톱에서 타이포와 간격을 키운 뒤에도 카드 내부 오버플로가 급증하지 않아야 한다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 피드 첫 화면에서 카드 소비를 시작하는 데 필요하지 않은 면책 및 설명성 텍스트를 콘텐츠 아래쪽으로 재배치하거나 기본 시야 밖으로 내려야 한다.
- **FR-002**: 시스템은 카드 성격을 반복 설명하는 소개 문구를 기본 화면에서 제거해야 한다.
- **FR-003**: 시스템은 마켓 컨텍스트 영역에서 실제로 읽을 콘텐츠가 없는 항목을 노출하지 않아야 한다.
- **FR-004**: 시스템은 모바일 헤더와 상단 액션 영역이 화면 폭을 안정적으로 사용하도록 구성해야 하며, 버튼과 텍스트가 비정상적으로 치우쳐 보이지 않아야 한다.
- **FR-005**: 시스템은 카드 탐색 방법을 장문의 도움말에 의존하지 않고, 인터랙션 자체만으로 이해 가능하게 제공해야 한다.
- **FR-006**: 시스템은 카드 내부 텍스트가 단어 중간에서 끊기지 않도록 표시해야 한다.
- **FR-007**: 시스템은 카드 내부 텍스트가 잘리지 않고 전체 메시지를 확인할 수 있도록 보여줘야 한다.
- **FR-008**: 시스템은 카드 이미지와 텍스트가 분리된 조각처럼 보이지 않도록 카드의 초기 표시 상태를 제어해야 한다.
- **FR-009**: 시스템은 카드 배경 이미지가 텍스트 가독성을 해치지 않도록 이미지 존재 시에도 텍스트 우선의 대비를 유지해야 한다.
- **FR-010**: 시스템은 카드 외부 메타 정보의 중복 노출을 줄여야 하며, 제목·카드 수·분류 정보는 사용자 이해에 필요한 최소 수준으로만 제공해야 한다.
- **FR-011**: 시스템은 정보 가치가 낮은 범용 태그를 기본 피드 경험에서 최소화해야 한다.
- **FR-012**: 시스템은 이슈 제목의 시각적 위계를 카드 자체보다 낮추고, 카드 외부의 캡션 또는 보조 설명에 가까운 수준으로 정돈해야 한다.
- **FR-013**: 시스템은 데스크톱 대형 뷰에서 카드 내부 타이포와 간격을 확장해 배경만 커져 보이는 인상을 완화해야 한다.
- **FR-014**: 시스템은 동일한 품질 문제를 반복 발생시키는 카드 생성 규칙을 식별할 수 있도록, 화면 문제와 생성 파이프라인 문제를 구분해 관리해야 한다.

### Key Entities

- **Feed Header**: 날짜, 이동 액션, 피드 진입 정보를 담는 상단 영역. 카드보다 앞서 보여도 되는 최소 정보만 포함한다.
- **Context Card Section**: 대표 지수·환율 등 보조 맥락을 제공하는 영역. 데이터가 있을 때만 보이며 카드 소비를 방해하지 않아야 한다.
- **Issue Feed Block**: 하나의 이슈를 구성하는 카드 묶음. 제목, 메타 정보, 태그, 카드 탐색 경험을 함께 포함한다.
- **Feed Card**: 사용자가 실제로 소비하는 핵심 단위. 텍스트와 이미지가 하나의 편집 결과물처럼 보이는 것이 중요하다.
- **Feed Chrome**: 면책, 안내, 태그, 카드 수, 버튼 라벨처럼 카드 외부의 보조 UI 집합.
- **Card Package Rule**: 한 이슈에 몇 장의 카드가 필요한지와 각 카드에 담길 텍스트 양을 결정하는 편집 규칙.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 피드 첫 진입 시 사용자는 스크롤 없이 카드 소비 진입부를 확인할 수 있고, 면책 문구는 첫 화면에 보이지 않는다.
- **SC-002**: 모바일 대표 뷰포트에서 상단 버튼과 텍스트 정렬 이상이 재현되지 않는다.
- **SC-003**: 데이터가 없는 마켓 컨텍스트 슬롯은 화면에 노출되지 않는다.
- **SC-004**: 카드 내 한국어 텍스트에서 단어 중간 줄바꿈이 시각 검수 기준으로 발견되지 않는다.
- **SC-005**: 카드 탐색 중 이미지와 텍스트의 어긋난 순차 노출이 시각 검수 기준으로 발견되지 않는다.
- **SC-006**: 카드 본문과 출처 정보는 말줄임 없이 확인 가능하다.
- **SC-007**: 피드 화면 내 설명성 문구 수가 현재 대비 유의미하게 줄어들고, 카드 이동은 별도 도움말 없이 수행 가능하다.
- **SC-008**: 카드 외부의 중복 메타 정보와 저가치 태그가 줄어들어, 사용자가 카드 자체에 집중할 수 있다.
- **SC-009**: 데스크톱 대표 뷰포트에서 카드 배경만 커져 보이는 인상이 완화되고, 내부 콘텐츠 가독성이 개선된다.
- **SC-010**: 제목과 메타 정보의 위계가 낮아져 카드 자체가 이슈 블록의 시각적 중심으로 인식된다.

## Assumptions

- 면책 문구는 완전히 제거 대상이 아니라, 정보 우선순위를 재조정해야 하는 대상이다.
- 마켓 컨텍스트는 핵심 카드보다 앞에 놓일 수 있지만, 실제 읽을 내용이 없으면 존재 자체가 품질 저하 요인이다.
- 사용자는 카드 탐색 방법을 학습하기보다 시각적 단서와 익숙한 인터랙션으로 이해하는 것이 바람직하다.
- 현재 화면 품질 문제 중 일부는 렌더링 문제이고, 일부는 카드 생성 규칙에서 비롯된다.
- 카드 외부 정보는 "없으면 치명적인 정보"만 남기고, 나머지는 최대한 카드 안으로 흡수하거나 제거하는 것이 기본 방향이다.

## Out of Scope

- 인스타그램형 피드 구조로의 전면 리디자인
- 카드 하단 캡션형 제목 체계로의 전면 전환
- 카드 생성 파이프라인 전반 재설계 없이 즉시 해결할 수 없는 편집 규칙 변경
- 데스크톱 대형 뷰에서 모든 카드 타입이 동일한 비율로 확대되도록 만드는 완전한 비례 스케일 시스템 재설계

## Follow-up Issues

- **후속 이슈 A - 카드 중심 피드 리디자인**: 이슈 제목을 카드 하단 캡션형 구조로 옮기고, 이슈 블록 자체를 이미지 중심 피드로 재배치하는 전면 리디자인
- **후속 이슈 B - 고급 반응형 카드 스케일링**: 대형 화면에서 모든 카드 타입이 동일한 비례 규칙으로 확대되도록 하는 시각 시스템 수립
- **후속 이슈 C - 가변 카드 패키징**: 이슈별 정보량에 따라 카드 수와 카드당 텍스트량을 동적으로 결정하는 생성 규칙 도입
- **후속 이슈 D - 카드 카피 편집 가드레일**: 말줄임, 긴 문장, 고유명사 분절이 발생하지 않도록 생성 단계의 텍스트 길이와 문장 구조 규칙 강화
