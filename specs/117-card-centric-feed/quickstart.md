# Quickstart: 카드 중심 피드 리디자인

## 검증 시나리오

### 시나리오 1 — 카드가 섹션을 지배하는지 확인

1. `npm run dev` 실행
2. 피드 페이지(`/feed/latest`) 열기
3. **확인**: 각 이슈에서 카드 스택이 시각적으로 가장 크게 보이는가
4. **확인**: 카드 위에 제목 텍스트가 없는가 (캡션은 카드 아래에만)
5. **확인**: 이슈를 감싸는 border/rounded 박스가 사라졌는가

### 시나리오 2 — 캡션 영역 정보 계층

1. 첫 번째 이슈의 카드 아래 캡션 영역 확인
2. **확인**: entityName이 작은 글씨로 최상단에 표시
3. **확인**: title이 그 아래 중간 크기로 표시
4. **확인**: changeValue(또는 entityType)가 작은 크기로 표시
5. **확인**: 공유 버튼이 캡션 우측에 있고 카드 위에는 없음

### 시나리오 3 — 이슈 간 구분

1. 여러 이슈가 있는 피드에서 스크롤
2. **확인**: 이슈 간 충분한 여백으로 구분됨 (선 없음)
3. **확인**: 카드에서 다음 카드로 자연스럽게 시선이 이어짐

### 시나리오 4 — 모바일 폭

1. Chrome DevTools → 375px 너비로 설정
2. **확인**: 카드 스택이 화면 너비의 90% 이상 활용

### 시나리오 5 — 데스크톱 폭

1. 1200px 이상 화면에서 피드 열기
2. **확인**: 카드가 단일 컬럼 중앙에 640px 내외로 표시
3. **확인**: 캡션 텍스트가 모바일보다 한 단계 크게 표시 (노안 배려)

### 시나리오 6 — 기존 인터랙션 유지

1. 카드 스와이프(모바일) 또는 이전/다음 버튼(데스크톱) 동작 확인
2. **확인**: 슬라이드, 진행률 표시, 버튼 모두 기존과 동일하게 작동

## 빌드 게이트

```bash
npx tsc --noEmit   # 타입 에러 없음
npm run lint        # 린트 통과
npm run test        # vitest 전체 통과
npm run build       # 프로덕션 빌드 성공
```
