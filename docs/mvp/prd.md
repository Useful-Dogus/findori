# 핀도리 (Findori) MVP PRD

> 문서 버전: v1.1  
> 최종 수정일: 2026-02-13  
> 문서 목적: 핀도리 MVP의 제품 범위와 요구사항을 정의하는 기준 문서

---

## 1. 제품 개요

### 1.1 제품 정의
핀도리(Findori)는 국내 주식 이슈를 카드형 콘텐츠로 요약해 제공하는 앱/웹 기반 정보 서비스다.

### 1.2 포지셔닝
- 정보 소비 미디어
- 투자 자문/추천 서비스가 아님

### 1.3 해결 문제
- 당일 이슈 파악에 시간이 오래 걸림
- 금융 정보의 이해 난이도가 높음
- 출처별 정보가 분절되어 맥락 파악이 어려움

---

## 2. 목표 사용자

### 2.1 1차 사용자
- 국내 주식에 관심이 있으나 긴 기사/리포트 소비 부담이 큰 개인 사용자

### 2.2 2차 사용자
- 특정 종목 중심으로 빠르게 이슈를 확인하고 공유하려는 사용자

---

## 3. MVP 범위

### 3.1 포함 범위 (In Scope)
- 앱/웹 동시 제공
- 장 마감 후 일일 카드 뉴스 발행
- 카드 피드 탐색: 세로(종목), 가로(카드)
- 채널 기반 카드 편집(최소 3개 채널)
- 종목 팔로우 및 우선 노출
- 인앱 알림(일일 이슈, 팔로우 종목 중요 이슈)
- 고유 URL 공유 및 비로그인 열람
- 카드별 출처 링크 제공

### 3.2 제외 범위 (Out of Scope)
- 투자 추천/매수·매도 시그널
- 포트폴리오 수익률 추적
- 댓글/커뮤니티 소셜 기능
- 해외주식/코인/부동산 카테고리

---

## 4. 핵심 사용자 흐름

### 4.1 진입
- 장 마감 후 인앱 알림 발송
- 알림 탭 시 당일 피드로 이동

### 4.2 소비
- 종목 단위 탐색 후 카드 순차 열람
- 카드 진행 상태 표시(예: 1/N)

### 4.3 공유
- 종목 이슈 단위 고유 URL 생성
- 수신자는 로그인 없이 열람 가능

---

## 5. 기능 요구사항 (FR)

### 5.1 콘텐츠 생성/발행
- FR-01: 장 마감 후 당일 카드 뉴스 자동 발행
- FR-02: 종목별 이슈를 채널 포맷에 맞게 카드로 구성
- FR-03: 모든 카드 뉴스에 대표 시각 요소 포함
- FR-04: 카드 마지막 장에 출처 및 액션(공유/팔로우) 제공

### 5.2 개인화
- FR-05: 사용자는 종목을 팔로우/언팔로우할 수 있어야 함
- FR-06: 팔로우 종목 콘텐츠를 피드 상단에 우선 배치
- FR-07: 팔로우 종목 이슈 발생 시 인앱 알림 발송

### 5.3 유통/접근
- FR-08: 공유 가능한 퍼머링크 제공
- FR-09: 비로그인 상태에서도 공유 콘텐츠 조회 가능
- FR-10: 앱/웹 간 동일 콘텐츠 제공

---

## 6. 콘텐츠 및 정책 요구사항

### 6.1 데이터 소스 범위
- 뉴스 미디어(포털 금융 뉴스, 주요 경제지, 경제 전문 매체)
- 증권사 공개 리포트
- 공개 커뮤니티/소셜 반응 데이터

### 6.2 신뢰/법무 기준
- 서비스 목적은 정보 제공이며 투자 자문이 아님
- 투자 판단 책임 고지 필요
- 카드별 출처 링크 제공
- 데이터 수집/활용 범위 고지 필요

---

## 7. 비기능 요구사항 (NFR)

- 모바일 우선 렌더링 최적화
- 앱/모바일 웹/데스크톱 웹에서 핵심 열람 기능 제공
- 공유 링크 진입 시 즉시 열람 가능한 응답성 확보
- 피드 탐색 중 체감 지연 최소화

---

## 8. MVP 완료 기준

### 8.1 제품 구현
- 앱/웹에서 동일한 당일 피드 열람 가능
- 인앱 알림 동선 정상 동작
- 채널 3종 이상 발행 가능
- 팔로우/우선 노출/중요 이슈 알림 동작
- 공유 링크 비로그인 열람 가능

### 8.2 콘텐츠/정책
- 카드별 출처 링크 누락 없음
- 투자 자문 오해 방지 문구 노출

### 8.3 초기 검증
- 지인 5명 내외 사용자 피드백 수집
- 피드백 기반 개선 항목 도출

참고: 정량 임계값(발행 수량, 중요 이슈 판단 기준 등)은 개발/운영 정책 문서에서 확정한다.

---

## 9. MVP 이후 검토 항목

- 채널 확장(신규 포맷)
- 검색/아카이브
- 종목 비교 시각화
- 개인화 추천 고도화
- 커뮤니티 기능 단계적 검토
