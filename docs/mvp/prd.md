# 핀도리 (Findori) MVP PRD

> 문서 버전: v2.0
> 최종 수정일: 2026-02-26  
> 문서 목적: 핀도리 MVP의 최소 제품 범위와 완료 기준을 정의한다.

---

## 1. 제품 개요

### 1.1 제품 정의

핀도리(Findori)는 국내 주식 시장 이슈와 대표 지수/환율 변동 원인을 슬라이드형 스토리로 구조화해 제공하는 모바일 우선 반응형 웹 브리핑 서비스다.

각 슬라이드는 하나의 독립 이슈를 다루며, 사용자는 스와이프 방식으로 순차 소비한다.

### 1.4 용어 및 상태 정합 기준

- 사용자 노출 최소 단위는 `이슈 카드`로 통일한다.
- 일일 발행 단위는 `이슈 카드 스트림`으로 통일한다.
- 상태 정의는 `loading`, `empty`, `error` 세트를 기준으로 한다.

### 1.2 포지셔닝

- 정보 소비 미디어
- 투자 자문/추천 서비스가 아님
- 뉴스 나열이 아닌 시장 해석 구조 전달 서비스

### 1.3 해결 문제

- 당일 이슈 파악에 시간이 오래 걸림
- 가격/지수 변동 이유를 빠르게 연결해 이해하기 어려움
- 출처별 정보가 분절되어 맥락 파악이 어려움
- 정보의 양은 많지만 인과 구조가 정리되어 있지 않음

---

## 2. 해결 전략

### 2.1 핵심 가설

- 시장 움직임은 "사건 → 변수 변화 → 자산 반응" 구조로 설명할 수 있다.
- 이 구조를 이슈 단위 슬라이드로 압축하면 이해 속도와 완주율이 개선된다.

### 2.2 제품 접근

- 하루 시장을 독립 이슈 단위로 분해한다.
- 각 이슈는 "사건 → 해석 → 시장 반응" 흐름으로 구성한다.
- 탐색보다 흐름 소비에 맞춘 스와이프 UX를 제공한다.
- 당일 핵심 흐름을 60초 내 이해 가능한 밀도로 설계한다.

---

## 3. 제품 철학 및 리스크 가드레일

### 3.1 뉴스와의 차이

- 뉴스: 사실 전달 중심
- 핀도리: 시장 반응의 인과 구조 전달 중심

### 3.2 투자 리스크 관리

- 매수/매도 조언 금지
- 목표가/수익률 약속 표현 금지
- 방향성 단정 표현 금지
- 서비스 목적은 판단 대행이 아니라 이해 보조

---

## 4. 목표 사용자

### 4.1 1차 검증 사용자

- 모바일 사용 선호 사용자 1명
- 데스크톱 사용 선호 사용자 1명

### 4.2 공통 니즈

- 오늘의 핵심 이슈를 빠르게 파악
- 특정 종목/지수/환율이 왜 변동했는지 구조적으로 이해
- 필요 시 원문 출처 확인

---

## 5. 사용자 경험 원칙

### 5.1 단위 통일

- 하나의 슬라이드는 하나의 이슈만 다룬다.
- 복합 주제를 한 슬라이드에 혼합하지 않는다.

### 5.2 정보 밀도 제한

- 짧은 텍스트 중심
- 명확한 인과 연결
- 감정적/선동적 표현 배제

### 5.3 소비 리듬

- 기본 소비는 좌우 스와이프 기반
- 진행 상태를 통해 완주를 유도
- 짧고 빠른 템포 유지

---

## 6. MVP 범위

### 6.1 포함 범위 (In Scope)

- 반응형 웹 제공(모바일/데스크톱)
- 장 마감 후 일일 이슈 카드 스트림 발행
- 이슈 카드 단위: 사건 → 해석 → 시장 반응 구조 제공
- 핵심 슬라이드에 변동 사실(수치/방향) + 변동 이유 요약 제공
- 대표 지수/환율 맥락 카드 제공(코스피, 나스닥, USD/KRW)
- 카테고리 태그 제공
- 이전 날짜 스트림 조회
- 고유 URL 공유 및 비로그인 열람
- 카드별 출처 링크 제공
- 투자 자문 아님 고지 노출

### 6.2 제외 범위 (Out of Scope)

- 투자 추천/매수·매도 시그널
- 포트폴리오 수익률 추적
- 댓글/커뮤니티 소셜 기능
- 개인화 추천
- 알림 기능
- 실시간 시세

---

## 7. 핵심 사용자 흐름

### 7.1 진입

- 홈 또는 공유 링크로 날짜별 스트림 진입

### 7.2 소비

- 슬라이드 스와이프로 이슈를 순차 소비
- 각 이슈에서 사건 → 해석 → 시장 반응 흐름 확인
- 이슈 태그로 맥락 분류 확인
- 필요 시 출처 링크 열람

### 7.3 공유

- 이슈 단위 고유 URL 생성
- 수신자는 로그인 없이 동일 콘텐츠 열람

### 7.4 회고 소비

- 이전 날짜 스트림을 조회해 시장 내러티브를 비교

---

## 8. 기능 요구사항 (FR)

- FR-01: 장 마감 후 당일 이슈 기반 이슈 카드 스트림을 자동 발행
- FR-02: 각 이슈 카드는 "사건 → 해석 → 시장 반응" 구조를 제공
- FR-03: 대표 지수/환율(코스피, 나스닥, USD/KRW) 맥락 카드를 당일 이슈 카드 스트림에 포함
- FR-04: 스와이프 기반 이슈 카드 소비 흐름과 진행 상태 표시 제공
- FR-05: 공유 가능한 퍼머링크 제공 및 비로그인 열람 지원
- FR-06: 모든 이슈 카드에 최소 1개 이상 출처 링크 제공
- FR-07: 피드/공유 랜딩에 투자 판단 책임 고지 노출
- FR-08: 날짜별 스트림 탐색(이전 날짜 조회) 제공
- FR-09: 이슈 카테고리 태그 제공

### 8.1 상태/예외 공통 기준

- loading: 피드/공유 진입 시 데이터 준비 중 상태를 의미한다.
- empty: 선택한 날짜 또는 조건에 발행 데이터가 없는 상태를 의미한다.
- error: 피드 로딩 실패/공유 실패/출처 링크 실패 등 재시도가 필요한 상태를 의미한다.
- 예외 발생 시 사용자는 홈 또는 재시도 경로를 즉시 인지할 수 있어야 한다.

---

## 9. 콘텐츠 및 정책 요구사항

### 9.1 데이터 소스 범위

- 포털 금융 뉴스, 주요 경제지, 경제 전문 매체
- 증권사 공개 리포트
- 지수/환율 공개 시세 정보

### 9.2 신뢰/법무 기준

- 서비스 목적은 정보 제공이며 투자 자문이 아님
- 핵심 주장/수치는 대응 출처를 추적 가능하게 제공
- 투자 판단 책임 고지를 주요 지점에 노출
- 매수/매도/목표가 유도성 표현 금지

---

## 10. 비기능 요구사항 (NFR)

- 모바일 우선 렌더링 최적화
- 모바일/데스크톱 웹에서 동일 의미로 콘텐츠 열람 가능
- 공유 링크 진입 시 즉시 열람 가능한 응답성 확보
- 피드 탐색 중 체감 지연 최소화
- 당일 핵심 스트림 60초 내 소비 가능 밀도 유지

---

## 11. MVP 완료 기준

### 11.1 제품 구현

- 당일 스트림이 모바일/데스크톱 웹에서 정상 열람됨
- 이슈 슬라이드에서 사건 → 해석 → 시장 반응 흐름을 확인 가능
- 종목 이슈와 지수/환율 맥락 카드가 함께 제공됨
- 각 카드에서 변동 사실 + 변동 이유 + 출처를 확인 가능
- 카테고리 태그와 이전 날짜 조회가 동작함
- 공유 링크가 비로그인 상태에서 정상 열림

### 11.2 콘텐츠/정책

- 카드별 출처 링크 누락 없음
- 투자 자문 오해 방지 문구 노출
- 투자 유도성 표현(매수/매도/목표가) 미포함

### 11.3 검증

- 모바일 선호 사용자 1명, 데스크톱 선호 사용자 1명 피드백 수집
- 두 사용자 모두 "오늘 핵심 이슈"와 "변동 이유의 구조" 이해 가능 여부 확인
- 두 사용자 모두 스트림 완주 가능 여부와 완주 소요 시간 확인
- 피드백 기반 개선 항목 도출

### 11.4 MVP 검증 목적

- 이번 MVP의 1차 목표는 정보 깊이 확장이 아니라 형식 검증이다.
- 즉, 이슈 구조화 포맷이 반복 소비 습관을 만들 수 있는지 확인한다.
