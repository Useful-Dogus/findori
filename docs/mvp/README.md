# Findori MVP Docs

MVP 문서 진입용 인덱스입니다.

## 30초 가이드

1. 범위를 확인한다: [`prd.md`](./prd.md)
2. 디자인 방향을 확인한다: [`design-strategy.md`](./design-strategy.md)
3. 구현 규칙을 확인한다: [`design-guidelines.md`](./design-guidelines.md), [`feature-spec.md`](./feature-spec.md)
4. 출시 전 점검한다: [`content-policy.md`](./content-policy.md), [`release-checklist.md`](./release-checklist.md)

## 문서 맵

- [`prd.md`](./prd.md): 무엇을 만들지 (범위, 요구사항, 완료 기준)
- [`design-strategy.md`](./design-strategy.md): 왜 이렇게 설계하는지 (원칙, 방향)
- [`design-guidelines.md`](./design-guidelines.md): 화면을 어떻게 만들지 (토큰, 컴포넌트, 상태)
- [`feature-spec.md`](./feature-spec.md): 기능이 어떻게 동작해야 하는지 (동작, 예외, QA)
- [`content-policy.md`](./content-policy.md): 콘텐츠 정책 (표현, 출처, 고지)
- [`release-checklist.md`](./release-checklist.md): 출시 점검 (Go/No-Go)

## 권장 사용법

- 기획 수정: `prd.md` 먼저 수정
- 디자인 수정: `design-strategy.md` 또는 `design-guidelines.md` 수정
- 기능 동작 수정: `feature-spec.md` 동기화
- 출시 판단: `release-checklist.md` 기준 사용
