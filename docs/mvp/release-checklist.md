# 핀도리 (Findori) MVP Release Checklist

> 최종 수정일: 2026-02-13  
> 목적: MVP 출시 전후 필수 점검 항목을 표준화한다.

## 1. 사전 조건

- [ ] 최신 PRD 기준 범위 동결 확인 (`/Users/chanheepark/dev/laboratory/findori/docs/mvp/prd.md`)
- [ ] 최신 디자인 전략/가이드라인 반영 확인
- [ ] 배포 환경(앱/웹) 접근 권한 및 설정 점검 완료

---

## 2. 핵심 기능 점검

### 2.1 발행/피드
- [ ] 장 마감 후 당일 카드 뉴스가 정상 발행된다
- [ ] 세로 스크롤(종목)과 가로 스와이프(카드)가 의도대로 동작한다
- [ ] 카드 진행 상태(1/N)가 정확히 표시된다
- [ ] 마지막 카드에 출처/공유/팔로우 액션이 노출된다

### 2.2 개인화
- [ ] 팔로우/언팔로우가 정상 동작한다
- [ ] 팔로우 종목이 피드 상단에 우선 노출된다
- [ ] 재진입 시 팔로우 상태가 유지된다

### 2.3 인앱 알림
- [ ] 일일 알림이 발송된다
- [ ] 중요 이슈 알림이 조건에 맞게 발송된다
- [ ] 알림 탭 시 대응 피드/카드로 진입한다

### 2.4 공유/비로그인
- [ ] 종목 이슈 permalink가 생성된다
- [ ] 공유 링크가 비로그인 상태에서 열람된다
- [ ] 링크 미리보기(썸네일/제목/요약)가 정상 노출된다

---

## 3. 콘텐츠/정책 점검

- [ ] 카드별 출처 링크가 누락 없이 포함된다
- [ ] 출처 링크가 실제로 열리는지 확인했다
- [ ] 투자 자문 오해 방지 문구가 노출된다
- [ ] 금지 표현(투자 유도/선동/비하) 미포함 확인
- [ ] 이미지/카피가 정책 위반 없이 노출된다

---

## 4. 앱/웹 정합성 점검

- [ ] 동일 이슈가 앱/웹에서 같은 순서와 의미로 노출된다
- [ ] 채널 라벨/핵심 카피/출처 정보가 앱/웹에서 일치한다
- [ ] 플랫폼별 UI 차이는 허용 범위 내에 있다

---

## 5. 비기능/품질 점검

- [ ] 모바일에서 가독성(폰트/간격/터치 영역)이 확보된다
- [ ] 웹(모바일/데스크톱)에서 핵심 열람이 가능하다
- [ ] 로딩/빈 화면/오류 화면이 각각 준비되어 있다
- [ ] 공유 링크 최초 진입 시 체감 지연이 과도하지 않다

---

## 6. 계측/운영 점검

- [ ] 핵심 이벤트 로그가 수집된다 (feed_opened, card_swiped, follow_toggled, share_clicked 등)
- [ ] 오류 로그 수집/조회 경로가 준비되어 있다
- [ ] 운영 문의/이슈 대응 담당자(본인) 확인 완료

---

## 7. 소규모 파일럿 점검 (지인 5명)

- [ ] 테스트 대상 5명 내외 선정 완료
- [ ] 테스트 시나리오 배포(진입→소비→공유)
- [ ] 피드백 수집 폼/노트 준비 완료
- [ ] 핵심 질문 준비 완료
- [ ] 결과 정리 및 개선 백로그 등록 완료

핵심 질문 예시:
- [ ] 첫 화면에서 오늘 핵심 이슈를 바로 이해했는가
- [ ] 카드 흐름이 자연스럽고 피로하지 않았는가
- [ ] 공유받은 링크에서 맥락 이해가 가능했는가

---

## 8. Go/No-Go

Go 조건:
- [ ] 핵심 기능 블로커 없음
- [ ] 정책 위반 리스크 없음
- [ ] 파일럿 운영 준비 완료

No-Go 조건:
- [ ] 공유 링크 비로그인 열람 실패
- [ ] 출처 누락/고지 누락
- [ ] 앱/웹 정합성 중대 불일치
- [ ] 알림 동선 핵심 실패
