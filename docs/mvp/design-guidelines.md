## 2. 디자인 시스템 토큰

### 2.1 Color Tokens

Core:

- `--bg-primary: #0E1116`
- `--surface-1: #1A2230`
- `--surface-2: #212C3D`
- `--text-primary: #F2F5FA`
- `--text-secondary: #B2BDCC`
- `--border-default: #2A3445`

Semantic:

- `--positive: #18C47D`
- `--negative: #F45B69`
- `--neutral: #7DA2FF`
- `--warning: #F6B93B`

Usage rules:

- 상승/하락 색은 숫자와 아이콘에만 사용
- 텍스트 대비 4.5:1 미만 조합 금지

### 2.2 Typography Tokens

Font family:

- `Pretendard Variable`, `-apple-system`, `BlinkMacSystemFont`, `sans-serif`

Type scale:

- `--fs-xl: 24px / 1.3 / 700`
- `--fs-lg: 20px / 1.35 / 700`
- `--fs-md: 16px / 1.5 / 600`
- `--fs-sm: 14px / 1.5 / 500`
- `--fs-xs: 12px / 1.4 / 500`

Usage rules:

- 카드 헤드라인 최대 2줄
- 본문 텍스트 최소 14px
- 수치 텍스트는 본문보다 한 단계 크게 사용 가능

### 2.3 Spacing / Radius

Spacing:

- `--sp-2: 8px`
- `--sp-3: 12px`
- `--sp-4: 16px`
- `--sp-5: 20px`
- `--sp-6: 24px`

Radius:

- `--radius-sm: 10px`
- `--radius-md: 14px`
- `--radius-lg: 20px`

## 3. 레이아웃 가이드

### 3.1 Viewport 기준

- 모바일 우선 기준폭: `360-430px`
- 데스크톱 웹 콘텐츠 최대폭: `960px`
- 콘텐츠 좌우 마진 최소 `16px`

### 3.2 구조

- 상단: 날짜/피드 컨텍스트
- 본문: 카드 피드
- 하단: 공유/출처 액션

### 3.3 탐색 규칙

- 세로 스크롤: 그룹(종목/지수/환율) 이동
- 가로 스와이프: 동일 그룹 카드 이동
- 제스처 충돌 시 현재 축 잠금 적용

## 4. 핵심 컴포넌트 규칙

### 4.1 Insight Card

구성:

- 헤더: 엔티티명(종목 또는 지수/환율), 카드 진행(1/N)
- 본문: 변동 사실(수치/방향), 변동 이유(1~2문장)
- 푸터: 출처, 공유

규격:

- 카드 최소 높이: 뷰포트의 68%
- 카드 패딩: `20px`
- 섹션 간 간격: `12-16px`

행동 규칙:

- 카드 탭 확장 없음(스와이프 중심)
- 마지막 카드에 "출처 전체 보기" CTA 노출

### 4.2 Reason Block

- 제목: "왜 변했나"
- 본문 1~2문장 제한
- 핵심 수치가 있으면 출처 인라인 연결

### 4.3 Index Context Card

- 대상: 코스피, 나스닥, USD/KRW
- 항목별로 변동 사실/이유/출처를 같은 순서로 배치
- 항목 누락 시 "업데이트 지연" 상태를 명시

### 4.4 Share Button

- 클릭 즉시 공유 패널 호출
- 실패 시 링크 복사 fallback 제공

### 4.5 Source Link List

- 도메인명 + 제목 축약 형태
- 카드당 최소 1개 이상
- 외부 링크는 새 탭으로 열기

## 5. 카피라이팅 가이드

### 5.1 헤드라인

- 최대 22자 권장(모바일 2줄 내)
- 단정형 예측 문장 금지
- "무슨 변동"이 먼저 읽히게 작성

### 5.2 본문

- 이유 설명은 1~2문장
- 전문 용어 사용 시 짧은 풀이 제공
- 사실/해석/반응을 섞어 쓰지 않음

### 5.3 금지 표현

- 투자 유도형("지금 사야", "확정 수익")
- 공포/혐오/조롱 유도 표현
- 출처 없는 단정

### 5.4 고지 문구

- 예시: "본 콘텐츠는 투자 자문이 아닌 정보 제공 목적입니다."
- 노출 위치: 피드 상단, 공유 랜딩 상단

---

## 6. 상태별 화면 가이드

### 6.1 Loading

- 카드 스켈레톤 표시
- 로딩 중 레이아웃 점프 최소화

### 6.2 Empty

- 상황: 당일 발행 없음
- 규칙: 이유 1문장 + 재시도 CTA 1개

### 6.3 Error

- 상황: 피드 로딩 실패/공유 실패/출처 실패
- 규칙: 에러 요약 + 재시도 CTA + 대체 경로(홈/링크 복사)

---

## 7. 접근성 기준

- 텍스트 대비 4.5:1 이상
- 인터랙티브 요소 최소 `44x44px`
- 색상만으로 상태 전달 금지(아이콘/텍스트 병기)
- 이미지 요소 대체 텍스트 제공
- 키보드 탐색 시 포커스 링 명확히 표시

---

## 8. 구현 정합성 체크

반드시 동일해야 하는 항목:

- 카드 정보 구조(헤더/본문/푸터)
- 변동 사실/이유/출처의 배치 순서
- 공유 URL 진입 시 핵심 정보 순서

허용 차이:

- 모바일/데스크톱 레이아웃 밀도
- 시스템 공유 UI 스타일

---

## 9. 산출물/검수 체크리스트

디자인 산출물 필수 항목:

- 피드 메인 화면(모바일, 데스크톱)
- 카드 상태(기본/로딩/에러/빈)
- 지수/환율 맥락 카드
- 공유 랜딩 화면

구현 후 검수:

- 모바일/데스크톱 정보 구조 불일치 여부
- 접근성 기준 위반 여부
- 공유 동선 정상 여부
