## 3. 레이아웃 가이드

### 3.1 Viewport 기준
- 모바일 우선 기준폭: `360-430px`
- 웹 콘텐츠 최대폭: `720px`
- 콘텐츠는 중앙 정렬, 좌우 마진 최소 `16px`

### 3.2 구조
- 상단: 날짜/피드 컨텍스트 + 설정/프로필 액션
- 본문: 카드 피드
- 하단: 주요 액션(공유, 팔로우, 출처 보기)

### 3.3 스크롤/스와이프 규칙
- 세로 스크롤은 종목 카드 그룹 단위 이동
- 가로 스와이프는 동일 종목 카드 페이지 이동
- 수직/수평 제스처 충돌 시 현재 축 우선 잠금 적용

## 4. 핵심 컴포넌트 규칙

### 4.1 Stock Card (핵심 컴포넌트)
구성:
- 헤더: 종목명, 티커, 채널 배지, 카드 진행(1/N)
- 비주얼: 대표 이미지/그래픽
- 요약: 핵심 한 줄 + 보조 1~2줄
- 푸터: 출처, 공유, 팔로우

규격:
- 카드 최소 높이: 뷰포트의 72%
- 카드 패딩: `20px`
- 헤더/본문/푸터 간 간격: `12-16px`

행동 규칙:
- 카드 탭 시 상세 확장 없음(스와이프 중심)
- 마지막 카드에만 "출처 전체 보기" CTA 노출

### 4.2 Channel Badge
- 라벨 + 배경색 + 경계선 조합
- 배지는 카드 헤더 좌측 상단 고정
- 채널 라벨은 최대 8자

### 4.3 Progress Indicator
- 점(dot) 또는 분수(1/N) 중 한 방식 고정
- 동일 종목 내 카드 수가 바뀌어도 시각 규칙 유지

### 4.4 Follow Button
상태:
- `default`: 팔로우
- `active`: 팔로잉
- `busy`: 처리중

규칙:
- 카드 내 버튼과 종목 상세 버튼 상태 동기화
- 반응 지연 시 로딩 상태 500ms 이내 표시

### 4.5 Share Button
- 클릭 즉시 공유 패널 호출
- 공유 실패 시 재시도 액션 제공
- 웹에서는 링크 복사 fallback 제공

### 4.6 Source Link List
- 출처는 도메인명 + 제목 축약 형태
- 카드당 최소 1개 이상
- 외부 링크는 새 창/외부 브라우저로 열기

## 5. 채널별 표현 가이드

### 5.1 공통 규칙
- 모든 채널은 "핵심 사실"을 첫 카드에 배치
- 감정형 표현은 보조 문구에서만 제한적 허용
- 투자 판단을 지시하는 문장 금지

### 5.2 채널별 톤
- 속보: 짧고 명확, 숫자 중심
- 개미토크: 반응 요약 중심, 과장형 문장 제한
- 애널리스트: 데이터/근거 중심
- 스토리: 맥락 연결 중심
- 대결: 비교축 명확화
- 밈: 주목도 확보 목적, 정보 전달을 가리지 않도록 제한

---

## 6. 카피라이팅 가이드

### 6.1 헤드라인 규칙
- 최대 22자 권장(모바일 2줄 내)
- 단정형 예측 문장 금지
- "왜 중요한지"를 포함한 문장 우선

### 6.2 본문 규칙
- 문장 길이 2줄 이내 권장
- 전문 용어 사용 시 짧은 풀이 제공
- 사실/의견/반응을 섞어 쓰지 않음

### 6.3 금지 표현
- "지금 사야", "무조건", "확정 수익" 등 투자 유도형
- 공포/조롱/혐오를 유도하는 표현
- 출처 없는 확정 단정

### 6.4 고지 문구
- 서비스 주요 지점에 정보 제공 목적 고지 노출
- 예시: "본 콘텐츠는 투자 자문이 아닌 정보 제공 목적입니다."

---

## 7. 인터랙션 및 모션

### 7.1 원칙
- 모션은 정보 전환 이해를 돕는 수준으로 제한
- 과도한 바운스/패럴랙스 효과 금지

### 7.2 기본 모션
- 카드 전환: `180-240ms`, `ease-out`
- 버튼 상태 전환: `120-160ms`
- 피드 진입 페이드: `200ms`

### 7.3 햅틱/피드백
- 팔로우, 공유 성공 시 짧은 피드백 제공
- 실패 시 즉시 텍스트 피드백 + 재시도 버튼 제공

---

## 8. 상태별 화면 가이드

### 8.1 Loading
- 카드 스켈레톤 표시
- 로딩 중에도 레이아웃 점프 최소화

### 8.2 Empty
상황:
- 당일 발행 없음
- 팔로우 종목 없음

규칙:
- 빈 이유를 1문장으로 설명
- 다음 행동 CTA 1개 제공(종목 팔로우, 전체 보기)

### 8.3 Error
상황:
- 피드 로딩 실패
- 공유 실패
- 출처 링크 접근 실패

규칙:
- 에러 사유 요약 + 재시도 CTA
- 반복 실패 시 대체 경로 제공(웹 열기/링크 복사)

---

## 9. 접근성 기준

- 텍스트 대비 4.5:1 이상
- 인터랙티브 요소 최소 터치 영역 `44x44px`
- 색상만으로 상태 전달 금지(아이콘/텍스트 병기)
- 이미지 요소에는 대체 텍스트 제공
- 키보드 탐색(웹) 시 포커스 링 명확히 표시

---

## 10. 앱/웹 구현 정합성

### 10.1 반드시 동일해야 하는 항목
- 카드 정보 구조(헤더/비주얼/요약/푸터)
- 채널 배지 의미 체계
- 출처 노출 정책
- 공유 URL 진입 시 핵심 정보 순서

### 10.2 플랫폼별 차이 허용
- 내비게이션 UI 형태
- 공유 패널 호출 방식
- 시스템 토스트/스낵바 스타일

---

## 11. 산출물/검수 체크리스트

### 11.1 디자인 산출물 필수 항목
- 피드 메인 화면(모바일, 웹)
- 카드 상태(기본/로딩/에러/빈)
- 채널 3종 이상 샘플
- 공유 랜딩 화면
- 팔로우/언팔로우 상태

### 11.2 구현 전 검수
- PRD 범위 외 기능 포함 여부 확인
- 투자 유도 표현 포함 여부 확인
- 출처 노출 누락 여부 확인

### 11.3 구현 후 검수
- 앱/웹 시각/정보 구조 불일치 여부
- 접근성 기준 위반 여부
- 알림→피드→공유 동선 정상 여부

---

## 12. 변경 관리

- 토큰/컴포넌트 변경은 본 문서 업데이트 후 반영
- 예외 적용 시 사유, 영향 범위, 롤백 방법 기록
- 버전 업데이트 시 `문서 버전`과 변경 요약을 함께 기록
