## 실행 컨텍스트

**Git 상태:**
```
!`git log --oneline -10 && echo "---" && git status --short`
```

**열린 이슈 전체:**
```
!`gh issue list --state open --limit 30 --json number,title,labels,milestone,assignees | jq -r '.[] | "#\(.number) [\(.labels | map(.name) | join(","))] \(.title)"'`
```

**현재 열린 PR:**
```
!`gh pr list --state open --json number,title,headRefName,isDraft,reviews | jq -r '.[] | "#\(.number) \(if .isDraft then "[DRAFT]" else "" end) \(.title) -> \(.headRefName)"'`
```

**최근 머지된 PR (마지막 5개):**
```
!`gh pr list --state merged --limit 5 --json number,title,mergedAt | jq -r '.[] | "#\(.number) \(.title) (merged: \(.mergedAt[:10]))"'`
```

## 이 스킬의 역할

- 저장소와 GitHub 상태를 종합 분석한다
- 실행 가능한 작업 후보를 선별하고 순위를 매긴다
- 사용자 승인용 큐를 제시한다
- **구현하지 않는다. PR을 열지 않는다. 머지하지 않는다.**

## 사전조건 검증

다음 중 하나라도 실패하면 큐 생성을 중단하고 이유를 보고한다:

- `gh auth status` 인증 확인
- 기본 브랜치(main)가 안정적인 상태인가
- 실행 중인 CI 실패가 있어 전체 작업에 영향을 주는가

## 후보 분석 기준

각 이슈를 아래 기준으로 평가한다:

**포함 기준:**
- 의존 이슈가 완료되었거나 의존성 없음
- 열린 PR과 파일/기능 충돌 없음
- 제품 결정 없이 구현 가능한 범위
- 스펙 작성에 충분한 컨텍스트 존재

**제외/보류 기준:**
- 미해결 의존성
- 활성 PR과 겹치는 작업
- 불명확한 요구사항
- 기반 CI 실패
- 누락된 권한

## 후보 점수 기준

각 후보를 아래 기준으로 점수화한다:

1. **크리티컬 패스**: 다른 이슈를 언블록하는가
2. **의존성 준비도**: 선행 작업이 완료되었는가
3. **범위 크기**: 작고 독립적인가
4. **검증 부담**: 테스트/검증이 단순한가
5. **머지 신뢰도**: 리뷰 통과 가능성
6. **불확실성**: 알 수 없는 부분이 얼마나 있는가

## 출력 형식

큐에 포함될 각 항목에 대해 보고:

```
### [순위]. #[이슈번호] [이슈 제목]
- **선택 이유**: [지금 이 작업인 이유]
- **의존성 상태**: [충족됨 / 없음 / 조건부]
- **예상 변경 규모**: [소 / 중 / 대]
- **주요 리스크**: [알려진 리스크]
- **머지 신뢰도**: high / medium / low
```

그 다음 보류/제외 항목 요약:

```
### 보류 항목
- #[번호] [제목] — [보류 이유]

### 제외 항목
- #[번호] [제목] — [제외 이유]
```

마지막으로 다음 문구로 마무리:
```
---
큐 제안이 완료되었습니다. 구현은 아직 시작되지 않았습니다.
위 큐를 승인하시면 /ship-next-loop 로 실행할 수 있습니다.
수정이 필요하면 순서 변경, 제외, 추가를 알려주세요.
```

## 큐 규칙

- 보통 3~7개 항목으로 구성 (너무 많으면 실용성 저하)
- 크리티컬 패스 언블로커를 코스메틱 작업보다 우선
- 수락 기준이 이미 명확한 이슈 우선
- 열린 PR이나 미해결 설계 결정과 충돌 가능성이 있는 항목은 큐에 넣지 않음

## ship-next-loop 핸드오프 준비

사용자가 큐를 승인하면 다음 정보를 `/ship-next-loop`에 전달할 수 있도록 정리한다:

- 승인된 이슈 번호 순서 목록
- 사용자가 변경한 우선순위나 제외 항목
- 사용자가 지정한 중단 조건 (없으면 "큐 소진 시 중단")
