## 이 스킬의 역할

`/ship-next-pipeline`은 얇은 오케스트레이션 레이어다:

1. `/ship-next-queue` 실행 → 큐 제안
2. 사용자 승인 대기 (필수 게이트)
3. 승인된 큐를 `/ship-next-loop`에 전달 → 반복 실행

**이 스킬은 큐 순위 로직이나 루프 실행 로직을 직접 구현하지 않는다.**

## 파이프라인 상태

아래 중 현재 상태를 항상 명시한다:

- `[큐 생성 중]` — ship-next-queue 실행 중
- `[승인 대기]` — 큐 제안 완료, 사용자 응답 기다리는 중
- `[큐 실행 중]` — ship-next-loop 실행 중
- `[블로커로 중단]` — 루프가 블로커를 만나 중단됨
- `[완료]` — 큐 소진 또는 사용자 정의 중단 조건 충족

## 단계 1: 큐 생성

`/ship-next-queue` 동작을 수행하여 저장소와 GitHub 상태를 분석하고 실행 가능한 큐를 제안한다.

큐 생성에 실패하면 실행 단계로 진행하지 않는다.

## 단계 2: 승인 게이트 (필수)

큐 생성 후 반드시 멈추고 사용자에게 다음 옵션을 제시한다:

```
파이프라인 상태: [승인 대기]

위 큐를 실행하시겠습니까?
- 승인: "승인" 또는 "진행"
- 순서 변경: 원하는 순서로 이슈 번호를 다시 나열
- 항목 제외: "제외: #N" 형식으로 지정
- 중단 조건 추가: "N개 완료 후 중단" 등
- 취소: "취소" 또는 "중단"
```

**승인 없이 구현을 시작하지 않는다.**

승인이 모호하거나 큐 내용을 변경하는 조건부 승인이면, 먼저 해당 사항을 해결한 뒤 진행한다.

## 단계 3: 승인된 큐 전달

사용자가 승인하면 정확히 승인된 내용을 `/ship-next-loop`에 전달한다:

- 승인된 이슈 번호 순서
- 사용자가 변경한 우선순위나 제외 항목
- 사용자가 지정한 중단 조건

**실행이 시작된 후에는 새로운 큐를 자동으로 재발견하지 않는다.**

## 단계 4: 루프 실행 감시

`/ship-next-loop` 실행 중 파이프라인 상태를 계속 보고한다.

루프가 블로커로 중단되면:
- 정확한 블로커 내용을 보고한다
- 하위 스킬의 상태를 보존한다
- 사람이 처리해야 할 후속 작업을 명확히 안내한다

## 실패 처리

| 상황 | 처리 |
|------|------|
| 큐 생성 실패 | 실행 진행 없이 중단 및 보고 |
| 사용자가 큐 미승인 | 부작용 없이 중단 |
| 루프가 승인 후 실패 | 정확한 루프 블로커 보고, 하위 스킬 상태 보존 |
| 승인이 모호하거나 조건부 | 명확화 후 진행 |
